## 1. 基本戦略：差別化特徴量への集中

### 設計思想

```
KeibaCICDは「全情報を詰め込んで精度を上げる」モデルではない。

ターゲット:
  「市場（オッズ）が見落としやすい情報」に特化した
  独自の視点を持つモデル

差別化の源泉:
  一般モデルが使う情報 → 市場に既に織り込まれている
  KeibaCICDが独自に評価する情報 → 市場に織り込まれていない
  → この差がEV（期待値）の源泉になる
```

### 特徴量の3分類

```
【A: 独自評価特徴量】← 育てる対象
  市場が見落としやすい、または評価しにくい情報
  例: 成長トレンド、リフレッシュ効果、コース適性の深掘り
  → 積極的に追加・チューニング

【B: 補完特徴量】
  市場も見ているが精度向上に貢献する情報
  例: 騎手・調教師統計、ペース適性
  → 効果を検証しながら取捨選択

【C: 市場情報（MARKET）】← 意図的に除外
  オッズ・人気など市場が直接見ている情報
  → 独自モデル（VALUE）からは除外
  → 市場との乖離を測るために使用
```

### FH-001: 前走人気（ROTATION）

**仮説**
「前走より人気が大幅に落ちた馬は、市場が過剰に評価を下げており、実力との乖離が生まれやすい」

**競馬的背景**
- 馬券購入者は直近の結果に過剰反応する傾向がある
- 前走惨敗でも原因が明確（距離ミス・馬場ミス等）なら実力は変わらない
- 市場はこの「原因」を正確に評価できない

**実装方法**
- 計算式: prev_popularity - current_popularity（正=人気落ち）
- データソース: JRA-VAN RACE_SYUTSUBASHIRA

**検証結果**
- 状態: 採用
- 重要度順位: 1位（勝利独自モデル）
- 追加日: v5.x以前

---

### FH-002: 出走頭数増減

**仮説**
「多頭数から少頭数へ、または少頭数から多頭数への変化は、馬の実力評価に影響するが市場はこれを正確に調整しない」

**競馬的背景**
- 少頭数レースでは着順が改善しやすい
- 市場は「頭数の変化」より「絶対的な着順」を見る傾向がある

**検証結果**
- 状態: 採用
- 重要度順位: 10位（勝利独自モデル）

---

### FH-003: 着順標準偏差（5走）

**仮説**
「着順のばらつきが大きい馬（一発型）は、オッズが安定型と同じ計算で付けられており、爆発時に割安になりやすい」

**競馬的背景**
- 市場は期待値より「安定性」を評価する傾向がある
- 一発型の馬は平均着順が悪く見えるため人気が落ちやすい

**検証結果**
- 状態: 採用
- 重要度順位: 8位（勝利独自モデル）

---

### FH-004: 近走成績トレンド

**仮説**
「直近の成績が上昇トレンドにある馬は、市場が過去の低成績を引きずって評価しており、実力との乖離が生まれやすい」

**競馬的背景**
- 市場は過去の成績を加重平均的に評価する
- 上昇途中の馬はまだ低評価期間の成績が引きずっている

**検証結果**
- 状態: 採用
- 重要度順位: 6位（勝利独自モデル）
- 将来: 成長曲線モデル（PR）の基盤になる

### FH-005: grade_level（未実装）

**仮説**
「クラスレベルをモデルが知ることで、絶対的な能力指数の精度が向上する」

**競馬的背景**
- G1馬と未勝利馬が同じスケールで評価されている現状の修正
- ARの精度向上に直結

**実装方法**
- MARKET除外からVALUEに移動
- grade_level: 新馬=1, 未勝利=2, 1勝=3, 2勝=4, 3勝=5, OP=6, G3=7, G2=8, G1=9

**検証結果**
- 状態: 未実装
- 優先度: Sprint 3（Phase B）
- 備考: ARの根本精度改善に必要

### FH-006: キャリブレーション補正（Sprint 2）

**仮説**
「WVモデルのpred_proba_wvをキャリブレーション補正することで、EVの精度が向上しROIが改善する」

**背景**
- 現在のWVモデルは約3倍の過小予測が確認されている
- EV = pred_proba_wv × odds の精度が低い状態

**実装方法**
- Platt Scaling または Isotonic Regression
- バックテストデータで補正パラメータを学習

**検証結果**
- 状態: 検証中（Sprint 2）

---

### FH-007: 血統成長パターン（長期構想）

**仮説**
「父・母父の血統によって成長のピーク時期が異なり、成長途中の馬は市場に過小評価される傾向がある」

**競馬的背景**
- 早熟系血統（サンデー系等）: 3歳春にピーク
- 晩成系血統: 4〜5歳でピーク
- 市場は現在の成績を見るが、ピーク前の馬は割安

**実装方法**
- 血統データベースのクラスタリング（早熟/晩成/標準）
- 年齢×血統の交互作用特徴量

**検証結果**
- 状態: 未実装
- 優先度: Phase C（成長曲線モデル）
- 参照: ability_rating_vision.md

## 6. 特徴量追加の判断基準

```
採用条件（全て満たすこと）:
  ① 仮説が「市場の見落とし」と整合している
  ② LightGBM重要度で上位50%以内
  ③ バックテストROIが+2pp以上改善
  ④ CI（信頼区間）の下限が100%を上回る

保留条件:
  ① 重要度は高いが統計的有意性が低い
  ② サンプル数が不足している

却下条件:
  ① 重要度が低い（下位25%）
  ② ROIへの影響がCI範囲内のノイズ
  ③ 市場情報に近い（EVの計算に循環が生じる）
```
