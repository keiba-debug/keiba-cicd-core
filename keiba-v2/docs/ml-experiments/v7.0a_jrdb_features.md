# v7.0a — JRDB IDM特徴量統合

## 概要

JRDB（競馬データベース）のSED（事後IDM）とKYI（事前IDM）データからML特徴量を計算し、
既存v6.2モデルに統合。JRDBが提供する高精度な能力指数を過去走統計として活用。

## 技術詳細

### データソース
- **SED index**: 286,060エントリ, 91.8MB（事後IDM、上がり/テン指数、出遅/不利補正）
- **KYI index**: 286,060エントリ, 108.7MB（事前IDM、各種予測指数）
- **結合キー**: `{ketto_num_10}_{race_date}` — 100%マッチ率

### v7.0→v7.0aのAUC急上昇について

`jrdb_jockey_idx` (重要度502K) が支配的すぎてモデルのアンサンブル多様性を阻害。
除外により他の特徴量が十分に学習され、汎化性能が大幅改善（+0.11 AUC）。
これは「特徴量支配効果」として知られる現象。

### ファイル
- `builders/build_jrdb_index.py` — SED/KYI indexビルダー
- `ml/features/jrdb_features.py` — JRDB特徴量計算モジュール（23特徴量）
- `ml/experiment.py` — JRDB統合、MARKET分類
- `ml/predict.py` — 推論パイプライン統合
- `ml/batch_predict.py` — バッチ推論統合
- `ml/optuna_tuner.py` — Optunaチューナー統合
