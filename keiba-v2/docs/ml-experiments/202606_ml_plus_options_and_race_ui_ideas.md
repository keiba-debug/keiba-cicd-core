# ML最終判断向けプラスオプション & レース表表示アイデア（横断まとめ）

作成日: 2026-06-11  
対象レビュー横断:
- RPCI
- IDM
- コース辞典
- 調教×調教師
- 出遅れ
- 血統
- 騎手接戦

---

## 1. 位置づけ（今回の狙い）

ここでの提案は、**ML本体の学習特徴量に即投入しない**前提で、  
「最終の買い目決定（bet_engine / 画面最終判断）で使う補助オプション」を整備するもの。

方針:
- AUC改善よりも `gap>=k` 条件下の ROI・下振れ抑制を優先
- 低信頼データは「強化要因」でなく「減点/警告」に使う
- UIでは「推奨」より「理由の可視化」を重視

---

## 2. 最終買い目決定で使えるプラスオプション案

## 2.1 リスクブレーキ系（買い目を減らす側）

### 1) 低信頼データ減点オプション

条件（例）:
- 指標の `sample_count` が閾値未満
- CI幅が大きい
- fallback使用中カテゴリ（IDM等）

使い方:
- `risk_penalty += w_low_confidence`
- 最終スコアが閾値未満なら見送り

効果:
- 「たまたま良く見える」シグナルでの誤購入抑制

### 2) PIT未保証データの保守モード

条件（例）:
- cutoff未指定の血統指標
- 全期間集計しかない分析値

使い方:
- 該当オプションは「参照のみ」にして stake 係数を縮小
- 高リスク券種（単勝厚張り）を禁止

効果:
- リーク起因の過信を運用で吸収

### 3) 出遅れリスク回避オプション

条件（例）:
- `horse_slow_start_rate` 高
- `jockey_slow_start_rate` 高
- 外枠/短距離/先行想定の組み合わせ

使い方:
- 先行前提ロジック時に減点
- 複勝中心へ券種シフト

効果:
- 取りこぼしではなく「そもそも発馬で終わる」負けの削減

## 2.2 強化系（買い目を増やす側）

### 4) 接戦勝ち切り優位オプション

条件（例）:
- `jockey_close_win_rate` が高水準
- 想定人気帯が拮抗（上位混戦）

使い方:
- 同スコア馬のタイブレークに加点
- 馬連/ワイド相手選びで優先

効果:
- 僅差決着での取り切り率向上

### 5) 調教×調教師パターン一致オプション

条件（例）:
- 当該調教師の高信頼パターン一致
- `lift` が正かつ `stability` 良好

使い方:
- 買い目候補の最終昇格トリガーに使用
- 「パターン一致 + ML優位」のみ強化

効果:
- MLスコア同等時の最終優先順位に差を作れる

### 6) コース事前バイアス整合オプション

条件（例）:
- 枠・脚質がコースバイアス方向に一致
- 開催進行（前半/後半）と整合

使い方:
- 本命の押し上げではなく、相手候補の選別で利用

効果:
- 相手選びの精度改善（特にワイド/馬連）

## 2.3 フィルタ連動系（ML結果との交差）

### 7) `gap>=k` 限定オプション

条件（例）:
- まずML主条件（`gap>=k`, EV>=x）を満たす
- その後、補助オプションで通過/除外

使い方:
- 補助オプションは単独発火させない
- 必ず主条件の後段で使う

効果:
- 補助指標の暴走防止

### 8) レースタイプ連動オプション（RPCI/ラップ）

条件（例）:
- 想定レースタイプ（瞬発/持続）と馬の適性差分が一致

使い方:
- 予測上位馬の中で並び替えに使用
- 不一致が強い馬は頭固定を避ける

効果:
- 展開ミスマッチ馬の過信を抑制

## 3. レース表画面に出すと有効な表示アイデア

## 3.1 1頭ごとの「最終判断カード」

表示項目（例）:
- ML主スコア（P/W/EV/gap）
- 補助オプション判定（+/-）
- 信頼度（データ品質）
- 最終判定（買い / 相手 / 見送り）

UI表現:
- 緑: 強化
- 黄: 注意
- 赤: 回避理由

## 3.2 理由タグ（Explainabilityバッジ）

例:
- `接戦◎`（騎手接戦勝率上位）
- `調教PT一致`（調教師パターン一致）
- `出遅れ注意`
- `低信頼データ`
- `血統cutoff未保証`

目的:
- 「なぜ買う/買わない」を一目で説明

## 3.3 リスクメーター（下振れ警報）

算出イメージ:
- 低信頼指標数
- 小標本指標比率
- PIT未保証指標の有無

表示:
- `Risk: Low / Mid / High`
- Highなら stake 自動抑制を提案

## 3.4 相手選び支援パネル

機能:
- 本命固定時の相手候補を補助指標順に並べる
- 「本命と真逆のリスク要因」を持つ馬を除外候補として提示

対象券種:
- 馬連 / ワイド / 三連系の相手選定

## 3.5 データ鮮度・信頼度ヘッダ

表示:
- 各分析データの `created_at`
- `sample_count` 閾値充足率
- fallback利用件数

目的:
- 古い分析を新しいレースに誤適用する事故を防ぐ

---

## 4. 実装の最小単位（すぐ着手できる順）

1. レース表に「理由タグ」を追加（まずは表示のみ）
2. 補助オプションを `bet_engine` 後段フィルタとして追加（主条件の後）
3. 低信頼データ減点を実装（CI/n/fallback）
4. `gap>=k` でオプションON/OFF比較ログを保存

---

## 5. 運用ルール（重要）

- 補助オプション単独で買い目を発火させない
- 強化より先に「回避ルール」を整備する
- 採用判定は AUCではなく ROI/CI/件数で行う
- 週次で「オプション別の寄与ログ」を見て重みを更新する

---

## 6. まず試す推奨3点

最初のスプリントはこの3つで十分:

- A. `低信頼データ減点`（全分析共通）
- B. `接戦◎` タイブレーク（同スコア馬の並び替え）
- C. `出遅れ注意` による先行馬の過信抑制

この3点は、学習再訓練なしで導入でき、  
買い目の下振れ抑制に寄与しやすい。
