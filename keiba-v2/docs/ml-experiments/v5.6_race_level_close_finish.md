# v5.6 前走レースレベル + 騎手接戦勝率

**Session 49** | 2026-02-25 | v5.5 → v5.6

## 概要

2つの新特徴量カテゴリを追加:
1. **前走レースレベル**: race_level_index.json を使って、前走の「メンバーレベル」を数値化
2. **騎手接戦勝率**: 1-2着間タイム差 0.1秒以内の接戦で勝ち切る力を数値化

## 背景・動機

- **前走レースレベル**: 「G1の5着」と「未勝利の5着」が同じ `avg_finish_last3` になる問題（ロードマップ A-3）
- **騎手接戦勝率**: ハナ差・クビ差の勝負を制する騎手の力量は、通常の勝率では表現しきれない

## 追加した特徴量

### 前走レースレベル（past_features.py に追加）

| 特徴量 | 説明 | データソース |
|--------|------|-------------|
| `prev_race_level_vs_class` | 前走レースのクラス内平均との乖離（連続値） | race_level_index.json |
| `avg_race_level_last3` | 直近3走のレースレベル平均 | 同上 |
| `prev_race_level_rank` | 前走レースランク (H=2, M=1, L=0) | 同上 |

- race_level_index.json は rating_standards.py が生成済み（9,303レース分）
- horse_history_cache の `race_id` フィールドでルックアップ
- MARKET_FEATURES には含めない（クラス内の相対レベルはオッズに直接反映されにくい）

### 騎手接戦勝率（jockey_features.py に追加）

| 特徴量 | 説明 | データソース |
|--------|------|-------------|
| `jockey_close_win_rate` | 接戦（0.1秒以内）での勝率 | race JSONから集計 |

- build_jockey_master.py に `build_close_finish_stats()` を新規追加
- 全20,501レースをスキャン → 7,309レース（35.6%）が接戦
- 204騎手に接戦データあり
- 接戦10回以上の騎手のみ有効値（それ以外は None → LightGBM NaN処理）
