# v5.6 EV vs Gap Analysis

## 概要
Session 45: EVをgapの代替/併用条件として検証する実験。
テスト期間拡張(test=2024-2026)とBootstrap CI分析を組み合わせ、
VB検出精度の向上を図る。

## 背景
- 現行VB: gap>=5 + margin<=1.2 → Win ROI 107.0%（783件, CI=[70-146%]）
- gap: モデル予測ランク - 人気ランク（順序情報のみ）
- 課題: 同じgap=6でもオッズ8倍と25倍を区別できない
- v5.4.1でIsotonicRegression導入済み → Win ECE=0.003でEV計算の信頼性が確保
- EV = calibrated_win_prob × win_odds

## 実験設定
- train: 2020-2022
- val: 2023
- test: 2024-2026（26ヶ月、テスト期間拡張）
- AUC: A=0.8231, B=0.7785, W=0.8405, WV=0.7828（2020-2023 trainと差は<0.006）
- EV閾値: 1.1, 1.2, 1.3, 1.5, 1.8, 2.0
- gap閾値: 5, 6, 7
- margin: 0.8, 1.2
- Bootstrap: 1000回レース単位リサンプリング、95% CI

## 主要な発見
1. **gap>=6+EV>=1.2+m<=0.8: CI下限102.9% > 100%** — 95% CIでROI>100%を初確認
2. **gapはEVより強い個別フィルタ** — gap>=6(+15,390) > EV>=1.2(+10,250)
3. **EVはsecondary filterとして有効** — gap>=6にEV>=1.2を追加でROI 147%→231%
4. **margin filterが最も重要** — gap>=5 m<=1.2(97.7%) vs m<=0.8(108.5%)で10pp差
5. **テスト期間拡張は有効** — CI幅-31%～-60%改善、AUC変化は<0.006
6. **gap 8-10帯がROI最大のスイートスポット** — 83件, ROI 215.2%, 利益貢献63%

## 注意点と限界
- CI下限>100%は115件の少数精鋭条件のみ。少数の高オッズ的中への依存リスク
- gap帯別分析(gap 8-10)とEV分析はPlace modelベース。bet_engine win_onlyはWin modelベース
- H1/H2 splitの利益偏りは統計的ノイズが主因と判断、ただしモデル劣化も否定不能
- 実運用でのスリッページ（オッズ変動）は未考慮
