# v5.12 学習期間拡大 + Point-in-Time化

> Session 58 | 2026-02-27
> Status: **DEPLOYED** as v5.12-pit3 (trainer/jockey PIT + snapshot fix)

## 背景

### 学習期間拡大 (Tier 0)
- 旧: Train 2020-2023 (12,400R) / Val 2024 / Test 2025-2026
- 新: Train 2020-2024 (16,650R, +38%) / Val 2025.01-02 / Test 2025.03-2026.02
- 月単位データ分割: `parse_period_range()` で "2025.01-2025.02" を解析

### Point-in-Time化 (A-4)
- 調教師/騎手の成績統計は全期間の累積値を使用 → テストデータの未来情報が漏洩
- 血統(sire/dam/bms)統計も同様にbuild_sire_stats.pyが全期間を使用

## 重要な発見

### 1. 母集団レベル vs 個人レベルの漏洩
- **個人レベル (trainer/jockey)**: 特定個人の成績累積 → PIT化で適切に漏洩除去
- **母集団レベル (sire/dam/bms)**: 数万頭の産駒実績 → PIT化でノイズ増大、効果なし

### 2. Snapshot ordering の重要性
```
# WRONG (pit1): snapshot → update → dates[i]にはi日目の開始時点を保存
# bisect_left(dates, race_date) - 1 → race_date-1日の開始時点（= race_date-2日の終了時点）

# CORRECT (pit3): update → snapshot → dates[i]にはi日目の終了時点を保存
# bisect_left(dates, race_date) - 1 → race_date-1日の終了時点（= race_dateの直前まで）
```

### 3. Model W が pit3 で base を上回った
- AUC 0.8397 (base 0.8380, +0.0017)
- 勝利予測はtrainer/jockeyの未来漏洩に最も敏感だった可能性
