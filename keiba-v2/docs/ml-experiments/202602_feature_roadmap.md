## 重要な設計原則

### 1. 解釈可能性を最優先する

```
❌ 現状: 特徴量 → LightGBM → gap/EV → 買い目（ブラックボックス）

✅ 目指す姿:
レースパターン × 馬の能力指数（± 未知数度）
    ↓
市場乖離スコア（説明可能）
    ↓
解釈可能な推奨理由付き買い目
    ↓
当日パラメータで人間が微調整
```

「なぜこの馬を推すか」が説明できることで、モデルの失敗理由の分析と特徴量改善が加速する。

### 2. 未知数度を明示する

同じ能力指数でも信頼区間の幅が異なる：

```
古馬OP実績10戦: 能力指数 62 ± 2（狭い → 点推定で判断）
新馬・未勝利:   能力指数 55 ± 8（広い → 幅で判断、大穴候補）
```

未知数度が高い馬は「期待値は低いがバリューが大きい」特性を持ち、市場が合理的に評価できない領域でもある。

### 3. 当日外部条件を人間が補完する

モデルは前走までのデータを使う。当日に発生する変数は人間が判断：

- 馬場状態（良/稍重/重/不良）× コース適性
- 枠順（当日発表の内外バイアス）
- 騎手変更（当日発表）
- パドック評価（主観だが有効）
- 当日の気温・湿度（芝の硬さ）

### 4. 再学習しない勇気

v5.7実験で判明：**再学習そのものがROIに±20ppのノイズを発生させる。**

CI幅が±15pp程度に縮小するまで（件数3,000件以上）、特徴量変更を伴う再学習は避ける。改善は「bet_engineの条件変更」と「フロントエンドの表示改善」に集中する。

### 設計方針

| ~~Selective~~ | ~~廃止~~ | ~~gap>=6, EV>=1.2, m<=0.8~~ | — | — |

### おいしさ表示

buy/don't buyの二値判断ではなく、参考情報として★スコアを表示：

```
★★★ gap>=8 + m<=1.2             → 大幅な市場乖離（ROI 215%帯）
★★  gap>=6 + m<=1.2             → Standard条件（CI下限>100%）
★   gap>=5 + m<=1.2             → Wide条件（件数重視）
```

> **EVは★条件から外した。** v5.8分析でEVの弁別力が弱いことが判明:
> EV>=1.5帯に85%の件数が集中（的中率4.7%）、EV 1.0-1.2帯の的中率16.7%が最高。
> EVはbet判断よりも「このbet の期待リターン」表示として参照用に残す。

Kellyサイジングは★の数に応じて傾斜（★★★は通常、★は半Kelly等）。

## margin filterの再解釈

v5.8分析（14ヶ月実績）の重要な発見：

**marginの効果はgap帯によって逆転する**

```
gap × margin クロス集計:
              m<=0.8    0.8-1.0    1.0-1.2
gap=5          46%       84%        149%   ← margin高い方がROI高い
gap=6          66%      294%        124%   ← 中間帯がスイートスポット
gap=7-8       232%        0%        200%   ← margin低い方がROI高い
gap=9+        162%      314%        402%   ← 全帯で好成績

理由（gap=5帯）:
margin低い（楽勝予測）→ 市場も同評価 → オッズ低い → ROI低い
margin高い（僅差予測）→ 市場と見解差 → オッズ高い → ROI高い

理由（gap=7-8帯）:
大幅な市場乖離がある馬は、margin低い（楽勝予測）でも
市場が過小評価している → オッズ十分 → ROI高い
```

margin filterは「一律の閾値」ではなく**gap帯に応じた使い方**が最適：
- gap=5: margin filterなしが最良（m<=1.2で十分）
- gap>=6: m<=1.2で十分（m<=0.8は利益を逃す）
- **結論: 全gap帯でm<=1.2が最適解** → 現行設計と合致

## 市場乖離スコアの研究（進行中）

### 現行のgapの限界

```
gap: モデル予測ランク - 人気ランク（順序情報のみ）

問題: 同じgap=6でも
・オッズ8倍の馬（市場の歪みは小さい）
・オッズ25倍の馬（市場の歪みは大きい）
を区別できない
```

### 新しい指標案

**① margin ÷ odds（市場乖離スコア）**
```
score = margin / odds
低いほど「モデル自信あり × 市場オッズ高い」= おいしい
```

**② 条件付きEV**
```
EV = calibrated_win_prob × odds
ただしEVが大きければ良いわけではない（EV>=1.5帯の的中率4.7%が示す通り）
```

**③ gap × margin × oddsの複合スコア**
```
VB_score = gap × (margin / odds)
ランク差・モデル確信度・市場配当を統合した単一スコア
```

→ 36頭の的中馬リストで検証予定

## 危険馬×VB複勝戦略（仮説）

> **前提課題**: Session 41で Place ROI < 100% が確認済み（Win-only > Win+Place）。
> 複勝ベースの戦略は、まず複勝単体のROIを100%以上にする方法を
> 見つけない限り実用化は困難。優先度を🟢低に設定。

### 仮説

```
危険な人気馬が存在するレース
    ↓
その馬が4着以下に凡走
    ↓
複勝配当が上昇（人気馬不在で配当が厚くなる）
```

### 検証計画

- Step 1: 「危険馬フラグON + 同レースVB馬あり」の条件で過去データを絞り込み
- Step 2: 複勝ROIをBootstrap CIで確認

注意点:
- 危険馬が好走した23%のケースでのVB馬複勝損失を確認すること
- 複勝単体のROI<100%問題を先に解決する必要あり

## 学びの蓄積（横断的）

### AUC改善 ≠ ROI改善
Model B精度向上が市場との乖離を縮めてVB効果を殺す。モデルを賢くするほど市場に近づく。

### 再学習そのものがリスク
importance=0の特徴量を除去してもROIが-18pp変動（v5.7実験）。AUC差<0.002でもROI差は18ppに達する。

### marginの逆説
margin filter（低い値）は的中率を下げる。市場も同じ評価をしているため。

### EVはsecondary filterとしてのみ有効
gap単独（+15,390円）> EV単独（+10,250円）。EVはgapの補助として使う。
さらにv5.8分析でEV>=1.5帯に件数の85%が集中し弁別力なしと判明。★条件から除外。

### marginの効果はgap帯で逆転する
gap=5ではmargin高い方がROI高い（市場と同じ評価→オッズ低い）が、
gap>=7ではmargin低い方がROI高い（市場が過小評価→オッズ十分）。
一律のmargin filterは避け、m<=1.2を全gap帯の共通上限とする。

### 測定基盤を先に整備
ROI改善を追う前に統計的有意性の判定基盤（Bootstrap CI）が必要。

### 累積P&Lは安定性の最終確認に有効
v5.8で3条件の14ヶ月累積P&Lを比較。条件B(gap>=6 m<=1.2)は
累積が一度もマイナスにならず（条件Aは-880円まで沈む）、DDも-80.7%に改善。
