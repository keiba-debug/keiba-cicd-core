# Phase 3b-6 Bayesian shrinkage キャリブ vs 手動 v0_2 決着

実施日: 2026-05-09
仮説: Bayesian shrinkage 付き auto-calibration なら手動 v0_2 を超えられるか?

## TL;DR

**Bayesian でも手動 v0_2 を超えられず**。OOS 2024-2025 (46R) で v0_4 は v0_3a (point) と完全同一の結果、v0_2 比 -1勝 (誤差範囲)。

| 評価セット | バージョン | top1 勝率 | top1 ROI | top3 ROI |
|---|---|---:|---:|---:|
| OOS 2024-2025 (46R) | polaris単独 | 23.9% (11/46) | 107.6% | 140.2% |
|  | **v0_2 (手動 +Phase 1 CI)** | **37.0% (17/46)** | **140.2%** | 117.2% |
|  | v0_3a (point OOS train) | 34.8% (16/46) | 140.9% | 115.8% |
|  | v0_3 (point in-sample) | 32.6% (15/46) | 131.7% | 107.3% |
|  | **v0_4 (Bayesian k=30)** | **34.8% (16/46)** | **140.9%** | (≈v0_3a) |

→ **手動 v0_2 が依然として最強** (1勝差は 46R サンプルでは誤差範囲)

## 1. Bayesian shrinkage 実装

`calibrate_rule_score.py` に `--method bayesian` を追加。

```python
prior strength k = 30
α₀ = k * overall_p       # ≈ 5.4 (overall_p ≈ 0.18)
β₀ = k * (1 - overall_p) # ≈ 24.6
posterior_mean = (α₀ + top3) / (k + n)
shrunk_logit = logit(posterior_mean) - logit(overall_p)
recommended = clip(shrunk_logit, [-1.0, +1.0])
```

特性:
- n=0 → posterior_mean = overall_p → shrunk_logit = 0 (effect なし)
- n=∞ → posterior_mean → top3/n (point estimate と一致)
- n=30 程度なら prior と同重み → 強い shrinkage

## 3. なぜ Bayesian でも v0_2 を超えられないか

仮説1: **Phase 1 検証時の手動 logit 値は既に CI ベース** (=暗黙の Bayesian)
- `_helpers.beta_ci()` で 3着内率の信頼区間を考慮
- 手動 v0_2 値は CI 下限ベースで保守的に設定済
- → Bayesian k=30 自動キャリブと同等の効果が手動で実現済

仮説2: **ドメイン知識込みの特殊条件が手動の優位性**
- `d2_naga_outer`: 永島まなみ×6枠以上 (n=0、auto-cal は何も学習できない)
- `d1_kikusawa_combo`: 菊沢一樹×菊沢隆徳 (n=15 でも +0.40 を Phase 1 で検証)
- `frame_7_late`: era 後期7枠 (era 分割の意図ある rule)
- → これらは手動でないと拾えない

仮説3: **OOS 46R はサンプル小**
- 17 vs 16 = 1勝差。Binomial 95% CI で見ると統計的有意ではない
- 真に同等の可能性が高い

## 4. 結論と運用方針

### 最終結論
- **手動 v0_2 と Bayesian v0_4 は OOS で実質同等** (1勝差は誤差)
- Bayesian は point estimate より明確に堅牢 (clipped 極端値が緩和)
- **特殊条件 (永島・菊沢コンビ等) を持つ v0_2 が僅差で勝つ理由はドメイン知識**

### Production 推奨
**v0_2 を採用** (手動値ベース)。auto-cal 結果は補助情報として活用:
- v0_4 と v0_2 のルール別差分を確認、明らかに方向性が違うルールは手動修正の検討
- 例: `bp_choku_master` は v0_2 +0.25 vs v0_4 -0.49 → 内枠の千直巧者は実は飛ぶ事実
  - v0_2 を `+0.05` 程度の弱い加点に下げる手動修正は検討余地あり

### 将来 (Phase 3b-7 もし必要なら)
- **ハイブリッド v0_5**: v0_4 ベース + 特殊条件 (n<30 のドメイン知識) のみ v0_2 から手動継承
- **win_rate 最適化**: top3_rate 最適化ではなく top1 単勝 win_rate を直接最適化する別キャリブ
- **複勝オッズ統合**: 払戻分散の小さい複勝での ROI 改善

## 5. 学び (重要、未来の自分に向けて)

1. **手動エビデンスベースのキャリブは過小評価しない**: Phase 1 verify の Bayesian CI 由来の保守性が OOS で勝った
2. **point estimate auto-cal は危険**: in-sample 過学習だけでなく、OOS train でも極端値が出やすい (clipped に張り付く rules 多数)
3. **Bayesian shrinkage は安全網として機能**: 極端値を防ぐが、手動値超えるほどの汎化能力は出ない
4. **OOS は train/test split で必ず**: in-sample 評価は楽観的すぎる
5. **小サンプルの特殊条件 (n<30) は手動維持が無難**: auto-cal は無情報か過学習に陥りやすい
