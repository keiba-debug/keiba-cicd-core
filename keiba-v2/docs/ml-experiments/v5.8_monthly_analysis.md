## 結論と二軸設計への示唆

### 現行条件の評価

| 条件 | 判定 | 理由 |
|---|---|---|
| win_only (gap>=5, m<=1.2) | **継続** | CI[85.9-193.2%], 月44.6件で実用的 |
| selective (gap>=6, EV>=1.2, m<=0.8) | **廃止推奨** | EVは弁別力なし、margin<=0.8が逆効果 |

### 新しい推奨軸の候補

**gap>=6 m<=1.2** が最有力:
- CI下限>100% (唯一)
- ROI 177.2%, 月25.4件
- gap=5帯を削ることで低ROI（46%@m<=0.8）セルを排除

### margin filterの再考

- **margin filterは「推奨度」ではなく「オッズとの関係」**
- 低margin（=モデルが「楽勝」と予測）→ 市場もそう見る → オッズ低い → ROI低い
- 高margin（=モデルが「僅差」と予測）→ 市場との見解差 → オッズ高い → ROI高い
- **marginを絞ることは、的中時のリターンを制限することと等価**
