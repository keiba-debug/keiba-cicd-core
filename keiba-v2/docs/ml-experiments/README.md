## 学び（横断的）

### 効果が大きかったもの
1. **JRA-VAN IDでの100%マッチ** (v3.0): AUC +0.03は最大の改善
2. **デュアルモデル設計** (v2): Value Bet戦略の基盤
3. **prev_race_popularity** (v3.1): 凡走予測仮説を実証、Model B重要度3位
4. **調教詳細データ統合** (v3.3): Model B AUC +0.0046、VB全ギャップでROI向上
5. **事前オッズ統合** (v3.4): データリーク解消しつつAUC維持
6. **3-way split + ECE** (v3.5): テスト汚染除去で真の性能を計測
7. **Win/Placeデュアル** (v5.0): 正確なEV計算を実現
8. **33ラップ** (v5.1): Win予測に特に有効 (prev_race_lap33=22,270)
9. **ベイズ平滑化 = MARKET** (v5.4): Place VB ROI 148.2%
10. **IsotonicRegression** (v5.4.1): Win ECE 0.13→0.003
11. **marginフィルタ** (v5.5): +13.9pp、最大の単発改善
12. **EV secondary filter** (v5.6): gap>=6+EV>=1.2でCI下限>100%初達成
13. **テスト期間拡張** (v5.6): train 1年短縮でCI幅-31%～-60%改善

### 期待ほど効かなかったもの
1. **脚質特徴量**: corners正規化だけでは脚質の本質を捉えきれない
2. **consumption_flag**: 発火条件が厳しく該当数が少ない
3. **CK_DATA調教 in Model B** (v4.1): AUC改善するがROI大幅悪化。MARKET扱いが正解
4. **余力ラップ8特徴量** (v5.2): ROI全gap帯で低下。ロールバック
5. **NLP辞書拡充** (v5.3b): 曖昧語追加でROI低下。少数精鋭が正解
6. **Pruning 20%一括** (v5.5 CI): Win VBを-27pp悪化
7. **Place VB ROI全般** (v5.5): bet_engine全プリセットでPlace ROI<100%。Win-onlyが最適
8. **EV単体フィルタ** (v5.6): gapより弱い。gapの補助としてのみ有効

### 特徴量エンジニアリング
- **MARKET/VALUE分類が最重要**: smoothed rateをVALUEにするとA-B乖離縮小→VB ROI低下
- **少数精鋭 > 大量追加**: コメントNLP辞書は精選版が優位
- **低importance ≠ 不要**: importance #71-91でもPlace VB +4.9pp改善可能
- **dead features(=0)のみ削除が安全**: is_koukaku_age, is_koukaku_handicap等

### モデル・手法
- **AUC改善 ≠ ROI改善**: Model B精度向上が市場との乖離を縮めてVB効果を殺す
- **IsotonicRegression**: Win ECE 0.13→0.003。EV計算にはcalibrated確率、ランキングにはraw
- **着差回帰 > 分類**: VB ROIでは回帰が全Gap帯で改善 (+4.6-9.8pp)
- **margin filter = 最大の単発改善**: gap>=5でmargin無し106%→margin<=1.2で119.9%

### 測定・評価
- **測定基盤を先に整備**: ROI改善を追う前に統計的有意性の判定基盤が必要
- **Bootstrap CI**: レース単位リサンプリング。馬単位は独立性の仮定が崩れる
- **テスト期間拡張が有効**: train 1年短縮(AUC<0.006差)でtest倍増 → CI幅-31%～-60%
- **gapはEVより強い単独フィルタ**: gap>=6(+15,390) > EV>=1.2(+10,250)
- **EV secondary filter**: gap>=6+EV>=1.2+m<=0.8で初のCI下限>100%
- **margin filterが基盤**: m<=0.8とm<=1.2で10pp差。最も安定した改善効果
- **累積P&L = CI補完の診断ツール**: 単調増加パターンは戦略の安定性を示唆

### ハイパラの知見
- `feature_fraction`: 特徴量増加時は上げる（0.7->0.8）
- Model B `num_leaves`: 市場系除外で情報量減のため、表現力を上げる（63->127）
- `min_child_samples`/`reg_lambda`: num_leaves増加時は過学習防止で上げる
- `scale_pos_weight=5.0` for Win models: is_win ~6.5% 不均衡対応
