## 学び

### 特徴量エンジニアリング
- **MARKET/VALUE分類が最重要**: smoothed rateをVALUEにすると A-B乖離縮小→VB ROI低下 (v5.4)
- **少数精鋭 > 大量追加**: コメントNLP辞書は精選版が優位 (v5.3 vs v5.3b)
- **低importance ≠ 不要**: importance #71-91でもPlace VB +4.9pp改善可能 (v5.3)
- **Pruningは保守的に**: importance下位20%一括削除はWin VBを-27pp悪化 (v5.5)
- **dead features(=0)のみ削除が安全**: is_koukaku_age, is_koukaku_handicap等

### モデル・手法
- **AUC改善 ≠ ROI改善**: Model B精度向上が市場との乖離を縮めてVB効果を殺す
- **IsotonicRegression**: Win ECE 0.13→0.003。キャリブレーション劇的改善
- **着差回帰 > 分類**: VB ROIでは回帰が全Gap帯で改善 (+4.6-9.8pp)
- **margin filter = 最大の単発改善**: gap>=5でmargin無し106%→margin<=1.2で119.9%

### 測定・評価
- **測定基盤を先に整備**: ROI改善を追う前に統計的有意性の判定基盤が必要
- **Bootstrap CI**: レース単位リサンプリング。馬単位は独立性の仮定が崩れる
- **366件のbet**: 95% CI幅110pp以上。バージョン間の小差は統計的に無意味
