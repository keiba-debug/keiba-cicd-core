## 概要

二値分類(is_top3, is_win)に加え、着差(time_behind_winner, 秒)を直接予測する
**回帰モデル(Reg B)** を追加。6モデル体制に拡張。

回帰モデルのpredicted_marginを**marginフィルタ**としてVB戦略に統合し、
Win ROI 106% → **119.9%（+13.9pp）** を達成。

さらに **bet_engine.py** をフロントエンドのJSON推奨として統合し、
4プリセット運用体制を確立。

|--------|---------|--------|---------|------|
| **Reg B** | target_margin (秒) | 92 (市場除外) | Huber (delta=2.0) | marginフィルタ |

### Reg Bの詳細

**target_margin**: 勝ち馬とのタイム差（秒）
- 1着: 0.0秒
- 2着: 0.1秒（0.1秒差）
- 最大: 5.0秒キャップ
- ソース: `TIME_SA` (JRA-VAN)、100%カバレッジ

**Huber損失**: 大きな外れ値（大差負け）にロバスト
```python
PARAMS_REG_B = {
    'objective': 'huber',
    'alpha': 2.0,  # delta
    'metric': 'l1',
    'num_leaves': 127,
    'learning_rate': 0.03,
    'feature_fraction': 0.8,
    ...
}
```

### Reg B メトリクス
| 指標 | 値 |
|------|-----|
| MAE | 0.7003秒 |
| 相関係数 | 0.6046 |
| BestIter | 742 |
| 特徴量数 | 92 |

## marginフィルタ

### 概念
`predicted_margin` = Reg Bが予測する「勝ち馬とのタイム差」。
小さいほど「勝ちに近い」ことを意味する。

VB戦略にmarginフィルタを追加:
```
買い条件 = (gap >= 閾値) AND (predicted_margin <= 閾値)
```

### gap × margin グリッドサーチ結果（均一100円買い、test=2025-2026）

| gap \ margin | <=0.6 | <=0.8 | <=1.0 | <=1.2 | <=1.5 | 制限なし |
|-------------|-------|-------|-------|-------|-------|---------|
| >=3 | 61.3% | 80.6% | 89.3% | 87.3% | 85.1% | 86.4% |
| >=4 | 60.3% | 90.4% | 93.1% | 90.1% | 87.8% | 89.2% |
| >=5 | 48.2% | 93.3% | 107.7% | **123.8%** | 113.0% | 108.6% |
| >=6 | 42.4% | 116.2% | **141.9%** | 134.3% | 130.2% | 120.2% |

**最適: gap>=5, margin<=1.2 → ROI 123.8%**（均一買い基準）

### margin閾値の比較（gap>=5, Kelly sizing）
| margin閾値 | ROI | 件数 | 純利益 |
|-----------|------|------|--------|
| <=1.0 | 107.7% | 少 | - |
| **<=1.2** | **119.9%** | 364 | **+7,260** |
| <=1.5 | 113.0% | - | - |
| 制限なし | 106.0% | - | - |

**margin<=1.2が最適**: 1.0は厳しすぎ（件数少）、1.5は緩すぎ（低品質混入）。

## bet_engine設計原則

### Win = rule-based（gap + margin）
- Win モデルはECE補正後でも0.003レベル → Kelly計算は可能だが安定性に疑問
- gap（ランク乖離）+ margin（着差予測）の複合条件が最もROI安定
- Kelly sizingは用いるが、エントリー判定はrule-based

### Place = EV + Kelly（参考）
- Place モデルのECEは元々良好 → Kelly最適化が理論的に有効
- ただし全プリセットでPlace ROI<100% → **現時点ではPlace不推奨**

### 均一買いROI ≠ Kelly ROI
- verify結果(均一100円): gap>=5, margin<=1.2 → **123.8%**
- bet_engine(Kelly sizing): 同条件 → **119.9%**
- Kelly Place betsが大きくなり全体ROIを低下させるケースがある

## 回帰 vs 分類: VB ROI比較

| Gap | Cls B ROI | Reg B ROI | 差分 |

## 学び

### 1. margin filter = 最大の単発改善
gap>=5でmargin無し106% → margin<=1.2で119.9%（+13.9pp）。
着差予測は市場乖離検出の精度を大幅に向上させる。

### 2. Win-only > Win+Place
win_only純利益(+7,260) > 最適Win+Place(+4,980)。
Place ROI<100%が全体を引き下げる。Place戦略は抜本的見直しが必要。

### 3. 着差回帰はランキングではなくフィルタとして使う
NDCG/AUCは分類が優位だが、VB ROIは回帰が優位。
「勝ちに近い」VB候補を選別するフィルタとしての利用が最適。

### 4. Huber損失の選択理由
大差負け（5秒+）の外れ値にロバスト。MSEだと外れ値に引きずられ、
MAEだと勾配不連続。Huber(delta=2.0)が両者の折衷。
