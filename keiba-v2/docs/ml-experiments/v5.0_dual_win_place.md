# ML v5.0: Win/Placeデュアルモデル + 正確なEV計算

**日付**: 2026-02-15
**種別**: モデルアーキテクチャ

---

## 概要

従来の2モデル体系（Model A/B、共にis_top3目的変数）を4モデル体系に拡張。
`is_win`（1着）を目的変数とするModel W/WVを新設し、正確なEV計算を実現。

### 重大発見
- **旧EV計算は理論的に不正確だった**: `pred_proba_v * winOdds` = P(top3) * 単勝オッズ
- 既存の Model A/B は両方とも `is_top3`（3着以内）が目的変数
- `is_win`（1着）列はデータセットに存在するがROI分析にのみ使用されていた

## 新モデル体系

| モデル | 目的変数 | 特徴量 | 用途 |
|--------|---------|--------|------|
| Model A | is_top3 | 全82 | 複勝精度 |
| Model V (旧B) | is_top3 | 69(市場除外) | 複勝バリュー |
| **Model W** | **is_win** | 全82 | **単勝精度** |
| **Model WV** | **is_win** | 69(市場除外) | **単勝バリュー** |

### 新EV計算

| 指標 | 旧 | 新 |
|------|-----|-----|
| 単勝EV | P(top3) * 単勝オッズ (不正確) | **P(win) * 単勝オッズ** (Model WV) |
| 複勝EV | - | **P(top3) * 複勝最低オッズ** (Model V) |
| 頭向き度 | - | **P(win) / P(top3)** |

## ハイパーパラメータ

Win models用の変更:
- `scale_pos_weight=5.0` for Win models（is_win ~6.5% 不均衡対応）
- 完全バランスは1/0.065=15だが、LightGBM AUC最適化は不均衡に強いため控えめに5.0
- それ以外のパラメータはv4.1と同一

## 学び

### 1. Win AUC > Place AUC
Model W AUC 0.8376 vs Model A 0.8243。1着は3着以内より予測しやすい。
勝ち馬には明確なシグナル（能力差が大きいケースが多い）がある。

### 2. Win ECEが高い（0.13）
`scale_pos_weight=5.0` がキャリブレーションを崩す。ただし `predict.py` の
レース内正規化（全馬のP(win)合計=1に調整）で実用上は問題ない。

### 3. Win VB ROIは低い
単勝はVB gap戦略だけでは利益が出ない（gap>=5で92.3%）。
単勝EVフィルタやトラック種別（芝/ダート）の複合条件が必要。

### 4. 複勝EVが新たに計算可能に
DB複勝オッズとP(top3)で正確な期待値が出せるようになった。
これにより単勝EV/複勝EV/頭向き度の3軸でベット判断が可能に。

### 5. Model Vイテレーション537
v4.1から変更なしだが、537回で早期停止。市場除外モデルは
学習に時間がかかるが過学習していない（valスコアが安定）。
