# ML Report 画面改善提案

## 背景

bet-engine.py 導入に伴い、ベット戦略が以下のように変更された:

| 項目 | 旧方式 | 新方式 |
|------|--------|--------|
| EV計算 | empirical ROIテーブル (gap固定) | Win=ルールベース / Place=calibrated確率×odds |
| 券種選択 | 芝=単勝, ダート=複勝 | Win/Place独立判断 (track type不問) |
| サイジング | Kelly (empirical確率) | Win=gap比例 / Place=Kelly (Model V確率) |
| フィルタ | gap のみ | gap + margin + EV(Place側) |
| 回帰モデル | なし | Reg B margin予測で追加フィルタ |

ML Report画面はモデル精度とVB戦略の分析ツールであり、
新方式のパラメータ判断・チューニングに使えるよう改善する。

## タブ別改善案

### 1. 概要タブ

#### 現状
- 4モデル (A, V, W, WV) の AUC / Brier / ECE を表示
- Top-N予測的中率

#### 改善

**1-1. ECEの意味を視覚化**

ECE値の横に信頼度インジケータを追加:

```
Model WV  ECE: 0.1231  🔴 Kelly不適格
Model V   ECE: 0.0054  🟢 Kelly適格
Model W   ECE: 0.1339  🔴 Kelly不適格
Model A   ECE: 0.0045  🟢 Kelly適格
```

判定基準:
- ECE < 0.03 → 🟢 Kelly適格
- ECE 0.03-0.05 → 🟡 注意
- ECE > 0.05 → 🔴 Kelly不適格

bet-engine が Win側で確率を使わない理由が一目でわかる。

**1-2. 回帰モデルセクション追加**

Place/Win Models の下に "MARGIN MODELS (着差回帰)" セクションを追加:

**1-3. bet-engine戦略サマリー追加**

概要タブの下部に、現在の bet-engine 設定を表示:

```
bet-engine 戦略 (standard)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
単勝: gap≥4 + margin≤1.2s → gap比例サイジング
```

### 5. 特徴量重要度タブ

#### 現状
- Model V のfeature importance（gain）バーチャート
- 4モデル切り替え可能

#### 改善

**5-1. 回帰モデルの特徴量重要度追加**

タブ切り替えに Reg A / Reg B を追加:

```
[Model A (Place精度)] [Model V (Place Value)] [Model W (Win精度)] [Model WV (Win Value)]
[Reg A (着差精度)] [Reg B (着差Value)]
```

回帰モデルの重要特徴量が分類モデルと異なるかを確認できる。
例えば回帰モデルで「前走3走平均着差」がより重要になっていれば、
着差回帰が着差固有のシグナルを捉えている証拠。

**5-2. 分類 vs 回帰 の重要度比較ビュー**

```
[比較モード: Cls B vs Reg B]

特徴量              Cls B順位  Reg B順位  差
直近3走平均着順       1位        3位      -2
騎手会場複勝率        2位        1位      +1
前走人気  ROTATION    3位        8位      -5
勝率厩舎指数          4位        2位      +2
...
```

順位差が大きい特徴量 = 回帰が分類とは異なる情報を使っている証拠。
