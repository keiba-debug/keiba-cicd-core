# Hypothesis Registry

**目的**: 仮説検証の事前登録（プリレジ）を行い、後付け解釈・多重検定バイアスを抑える。  
**運用開始**: 2026-06-11  
**関連**: `docs/ml-experiments/202606_lightgbm_market_gap_review.md`

---

## 1. 運用ルール

1. 検証前に本台帳へ仮説を登録する（実験後追加は禁止）。
2. 主要評価指標は最大2つ、副作用監視は4つ以内に限定する。
3. 採用/不採用基準は実験前に固定し、実験後に変更しない。
4. 期間Aで探索し、期間BまたはOOSで再検証する。
5. 低サンプル（原則 `n < 80 bets`）は観測扱いにし、採用判断しない。

---

## 2. ステータス定義

- `planned`: 登録済み・未実施
- `running`: 実施中
- `review`: 結果レビュー中
- `adopted`: 採用
- `rejected`: 不採用
- `on_hold`: 保留
