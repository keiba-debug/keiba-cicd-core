# v8.3 / polaris 2.3 — AR target 不利補正 (margin-mode adjusted)

## 概要

AR(着差回帰)モデルの target_margin に JRDB SED の不利補正値を適用する改善。
不利を受けて遅れた馬の真の能力を正しく評価することで AR 精度を向上させ、
副次的に P モデルにも好影響を期待する設計。

## 動機 (v8.2 失敗実験からの学び)

v8.2 (polaris 2.2 = AR-stack) で得た知見:
- AR と W の出力を「一致させる」方向は ROI 悪化を招く (Stars/Nebula独立性が崩壊)
- AR 単独の精度を上げる方向 (B案) は安全、Stars系の役割強化

→ AR target そのものを改善 (不利補正) して、Stars系の役割を保ったまま精度向上。

## 変更内容

### 1. CLIフラグ `--margin-mode` 追加

```bash
python -m ml.experiment --use-optuna --margin-mode adjusted --version 2.3
```

選択肢:
- `raw` (従来): `target = race_time - winner_time`
- `adjusted` (本採用): `target = (race_time - 0.1*furi_adj) - (winner_time - 0.1*winner_furi_adj)`
- `zscore`: レース内z-score化 (将来候補)
- `adj_zscore`: 不利補正 + z-score (将来候補)

### 2. JRDB SED 不利データの活用

JRDB SED index (Session 113 構築済) の `furi_adj` フィールドを使用:
- 値域: 0-11 の整数スコア (97.9% は 0)
- スケール: 0.1秒/単位 (推定、`--furi-scale` で調整可)

### 3. 補正対象

| Split | Total | 不利補正対象 | % |
|-------|-------|-----------|----|
| Train | 241,364 | 5,451 | 2.3% |
| Val | 3,624 | 57 | 1.6% |
| Test | 40,327 | 652 | 1.6% |

補正対象は少ないが、**「不利を受けた馬の能力評価」が変わることで AR出力分布全体がシフト**し、
P/Wモデルの予測にも間接的に影響する。

## 学び

### ✅ 達成

- **P Top1 単勝が黒字化** (+7,679, ROI 102.6%) — AR target改善が AR出力経由(`ar_deviation`等)で P 予測にも波及
- W Top1 勝率 +9.07pt (21.40 → 30.47%)、ROI も微増 (86.6 → 87.1%)
- 非favorite選択時の判断品質が大幅向上 (14.3 → 22.6%)
- AR 自体の精度も微改善 (MAE -0.0016s, Corr +0.0026)
- 2.2 で起きた「favorite寄り → ROI悪化」が ROI悪化なしで再発 (favorite寄りだが収益性は維持)

### ⚠️ トレードオフ

- W が favorite 寄りに変化 (人気3.29→1.67、fav一致30→63%)
  - AR-stack していないのに W が変化したのは、**AR出力経由で W特徴量が変化したため**(推定)
  - 5,451件の補正で AR 全体の予測スケールが微シフト
- W-VB(穴狙い) picks 半減 (4,056→1,818)、PnL -29,480 (黒字戦略消失)
  - ただし「過剰検出が厳選化された」とも解釈可能 — picks/レース 1.4→0.6
  - 真の穴は別モデル (Stars系) で検出する方針へ

### 設計原則の確認

- AR の精度を「単独で」上げる方向は安全、副次効果でPモデルも改善
- W との階層化 (v8.2 C案) は Stars/Nebula 独立性を破壊するため不可
- 穴馬検出は別モデル (Stars系の専用モデル) として並列構成にすべき
