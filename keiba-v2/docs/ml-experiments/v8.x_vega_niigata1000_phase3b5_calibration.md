## 1. キャリブ手法

設計書 §5.1 命中率差駆動:
```
condition_p = 条件成立時の 3着内率
overall_p   = 全体の 3着内率
rule_logit  = clip(logit(condition_p) - logit(overall_p), [-1.0, +1.0])
```

`calibrate_rule_score.py` がルール毎に matched_n / top3_rate / 推奨 logit を算出、`min_n=30` 未満は手動値維持。

詳細キャリブ表は `v8.x_vega_niigata1000_phase3b5_calibration_2020_2023.md` 参照。

## 3. なぜ手動値が OOS で勝つか

仮説: **Phase 1 verify は Bayesian CI ベースで暗黙の shrinkage をしていた**。
- `_helpers.beta_ci()` で 3着内率の信頼区間を計算
- CI が広い (サンプル少) ルールは控えめな logit_score を割り当て
- CI が狭い (サンプル多) ルールは強気の logit_score
- これは事実上の **partial pooling / Bayesian shrinkage**

一方 auto-calibration は単純な点推定:
- `condition_p = top3 / n` の不確実性を無視
- 結果としてサンプルバラツキを過剰反映 → OOS で劣化

## 4. 推奨アクション

### v0_4 設計案 (Bayesian shrinkage)
```
α, β = beta posterior parameters (prior: overall_p)
posterior_mean = (top3 + α) / (n + α + β)
shrinkage = max(0, 1 - 2 * sd(posterior_mean) / |logit_diff|)
rule_logit = shrinkage * (logit(posterior_mean) - logit(overall_p))
```

サンプルが少ないルールは shrinkage が強く効き、手動値に近い保守的な値になる。

### 当面の運用
- **production には v0_2 を採用** (OOS で実測 +13.1pt 勝率)
- v0_3a の知見は手動修正のヒントとして使う:
  - `bp_choku_master` の符号反転 → +0.25 を 0 もしくは小さい負値へ手動修正
  - `c3_old_horse` の強烈な負値 → -0.10 を -0.30 程度へ漸進的に増やす
- 自動キャリブの完全自動化は v0_4 (Bayesian) 待ち

## 5. 学び

1. **In-sample キャリブの過学習リスクは大きい**
2. **手動エビデンスベース (Phase 1 CI) は OOS 汎化に強い** (Bayesian shrinkage の暗黙効果)
3. **OOS は train/test split で必ず評価する** (今回は 2020-23 train, 2024-25 test)
4. **次回キャリブは Bayesian で**: 単純な命中率差は危険、posterior + shrinkage が必要
