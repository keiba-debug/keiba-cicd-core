## 概要

Win モデル (W/WV) のECEが0.13と非常に高い問題を IsotonicRegression で解決。
`scale_pos_weight=5.0` によるキャリブレーション崩壊を後処理で補正し、
全4分類モデルにキャリブレータを適用。

**特徴量の変更なし。** モデル自体は同一で、予測確率の事後補正のみ。

## 問題: Win ECEの高さ

| モデル | ECE (補正前) | 状況 |
|--------|-------------|------|
| Model A (Place) | 0.0046 | 良好 |
| Model B (Place) | 0.0054 | 良好 |
| Model W (Win) | **0.1339** | Kelly不適 |
| Model WV (Win) | **0.1231** | Kelly不適 |

Win モデルのECE 0.13は「予測確率が系統的にずれている」ことを意味し、
Kelly Criterionによる賭け額計算が不正確になる。

### 原因
`scale_pos_weight=5.0`がis_win(~6.5%)の不均衡を補正するが、
その副作用で予測確率がインフレする。

## IsotonicRegression

### 仕組み
```python
from sklearn.isotonic import IsotonicRegression

# validation set でキャリブレータを学習
ir = IsotonicRegression(out_of_bounds='clip')
ir.fit(pred_val_raw, y_val)

# test set に適用
pred_calibrated = ir.predict(pred_test_raw)
```

- **学習**: validation set (2024年) の予測確率と実際のラベルでフィット
- **適用**: test set (2025-2026年) の予測確率を補正
- **重要**: ランキング(順序)は不変。確率の絶対値のみ補正

### EV計算での使い分け
- **ランキング**: raw (生確率) — 順序不変なのでキャリブレーション不要
- **EV計算**: calibrated — 正しい確率でないとEV計算が狂う
- **Kelly計算**: calibrated — ECE良好が前提

## 結果

### ECE改善

| モデル | 補正前 | 補正後 | 改善率 |
|--------|--------|--------|--------|
| Model A | 0.0046 | **0.0030** | -35% |
| Model B | 0.0054 | **0.0059** | +9% (微悪化) |
| Model W | 0.1339 | **0.0039** | **-97%** |
| Model WV | 0.1231 | **0.0028** | **-98%** |

Win モデルのECE: 0.13 → 0.003 = **劇的改善**。

### AUC/Brier

| モデル | AUC | Brier (前) | Brier (後) |
|--------|-----|-----------|-----------|
| Model A | 0.8228 | 0.1279 | 0.1280 |

| Model W | 0.8371 | **0.0889** | **0.0573** |

- AUCは不変（キャリブレーションは順序を変えない）
- Win BrierがIsotonicで大幅改善（確率のスケールが正確に）

## 学び

### 1. IsotonicRegressionはWin modelに必須
scale_pos_weightによるECE崩壊を後処理で完全補正。
追加のモデル変更なしで、Kelly Criterionの前提条件をクリア。

### 2. valセットでfit、testに適用
キャリブレータ自体のデータリークを防止。trainは過学習、testは評価汚染になる。

### 3. raw vs calibrated の使い分け
ランキングにはraw（順序保証）、EV/Kellyにはcalibrated（確率精度保証）。
混同するとランキング品質またはEV計算精度が劣化する。
