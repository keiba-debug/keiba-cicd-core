# ML v5.2: 余力ラップ(ch7) + race_trendカテゴリ特徴量（+8特徴量）

**日付**: 2026-02-20
**種別**: 特徴量エンジニアリング
**結果**: **ロールバック推奨（v5.1に戻す）**

---

## 概要

ch7の余力ラップ理論を基に、レースのラスト1F・減速度から余力を検出する特徴量5個と、
v2分類のカテゴリ特徴量3個、計8個を追加。

余力ラップ系の上位3特徴量はModel B Top25に入ったが、8特徴量をまとめて追加した結果、
ECEが悪化しVB ROIが全gap帯で低下。v5.1にロールバックが推奨される。

## 新特徴量（8個）

| 特徴量 | 説明 | Model A | Model B |

| `fast_finish_top3_rate` | 速い上がりレースでの好走率 | ~800 | 917 (#64) |

## 判断: **v5.1にロールバック推奨**

### ロールバック理由
1. **AUCはほぼ横ばい**: 新特徴量は予測の順序精度に大きな影響を与えていない
2. **VB ROIが全gap帯で低下**: 複勝-3~-13pt、単勝(VB内)-2~-13pt
3. **ECEが3/4モデルで悪化**: キャリブレーション精度低下
4. **ROI低下幅がサンプリングノイズを超えている**: -13ptはシグナル的

### 唯一のプラス
- **Model WV ECEのみ改善**: 0.1315 -> 0.1228 (-0.0087)

### 次のアプローチ
- **上位3特徴量のみで再実験**: race_last1f_avg_last3 + prev_race_last1f + race_decel_l1f_avg_last3
- 低重要度の5つ（yoriki_score, trend_switch, dominant_trend, prev_trend, fast_finish）を除外して再検証

## 学び

### 1. 特徴量一括追加のリスク
8特徴量をまとめて追加するとノイズが増加する可能性。上位3つにシグナルがあっても
下位5つのノイズが相殺してしまう。小ロット（3-4個ずつ）で追加する方が安全。

### 2. race_trendカテゴリ系は低重要度
dominant/prev_trend_v2_enc は58-59位。整数エンコーディングでは
カテゴリ間の順序関係を正しく表現できていない可能性。

### 3. 余力ラップ上位3はシグナルあり
race_last1f(#16)、decel(#17)、prev_last1f(#21)。
ラスト1Fの物理データ自体は有用だが、他5特徴量と一緒に入れたことで効果が打ち消された。
