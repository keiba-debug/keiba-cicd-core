# v8.1 / polaris 2.0 — JRDB5種追加 + CID時系列特徴量

## 概要

JRDB新規5データ種(CYB/CHA/KKA/UKC/JOA)のパーサー構築・インデックス化を行い、
JOAに含まれるCID(コンディションインデックス)とLS(ロングショット)指数の追加が
P/W両モデルで歴代最大級の改善をもたらした。

さらにCIDの時系列特徴量（前走差分・平均乖離）を追加し、Wモデルの精度を追加改善。

### 新規特徴量20個 (features/jrdb_features.py)

**CYB (調教分析) — 5特徴量:**
- `jrdb_cyb_oikiri_idx` — 追切指数 (基準60)
- `jrdb_cyb_shiage_idx` — 仕上指数 (基準60)
- `jrdb_cyb_shiage_change` — 仕上指数変化 (1:++, 2:+, 3:不変, 4:-)
- `jrdb_cyb_training_eval` — 調教評価 (1:良, 2:可, 3:悪)
- `jrdb_cyb_oikiri_prev` — 一週前追切指数

**CHA (本追切) — 3特徴量:**
- `jrdb_cha_oikiri_idx` — 本追切の追切指数
- `jrdb_cha_shimai_idx` — 本追切の終い指数
- `jrdb_cha_oikiri_type` — 追切種類 (1:馬なり, 2:普通, 3:一杯)

**JOA (CID/LS) — 3特徴量:**
- `jrdb_cid` — CID (コンディションインデックス)
- `jrdb_cid_score` — CID素点
- `jrdb_ls_idx` — LS指数 (ロングショット)

**CID時系列 — 6特徴量:**
- `jrdb_cid_score_last` — 前走CID素点
- `jrdb_cid_score_avg3` — 直近3走CID素点平均
- `jrdb_cid_score_trend` — 今回 - 前走 (正=調子上向き)
- `jrdb_cid_score_vs_avg` — 今回 - 直近3走平均 (正=好調)
- `jrdb_ls_idx_last` — 前走LS指数
- `jrdb_ls_idx_trend` — 今回LS - 前走LS

**KKA (産駒成績) — 3特徴量:**
- `jrdb_kka_total_runs` — JRDB成績 合計レース数
- `jrdb_kka_win_rate` — JRDB成績 勝率
- `jrdb_kka_dam_rentai` — 母産駒最連対率(%)

### race_id → jrdb_race_key 変換ユーティリティ
`race_id_to_jrdb_key()`: KeibaCICD 16桁race_id → JRDB 8桁レースキー変換。

## 考察

### CIDが効いた理由
CID(Condition Index)は馬の調子（コンディション）を直接的に数値化した指標。
従来のJRDB特徴量(IDM, 総合指数等)は「能力」の評価だが、CIDは「状態」の評価。
能力×状態の掛け合わせが予測精度を大幅に向上させた。

### CID時系列の価値
`jrdb_cid_score_trend`（今回CID - 前走CID）がWモデル4位は、
「前走より調子が上がっている馬」が勝つ確率が高いことを直接捉えている。
これは人間の予想家が「上昇気配」と呼ぶ概念のデータ化に相当する。

### CYB/CHA調教系の寄与
CYB/CHAの調教指数はTop10に入らなかったが、これはKYIの`training_idx`と
`stable_idx`が既にカバーしている領域のため。ただし微量のノイズ低減に
貢献している可能性はあり、除外実験は未実施。

## 技術メモ
- `race_id_to_jrdb_key()`: YYYYMMDDJJKKNNRR → VVYYKHHR変換
- CYB/CHA/KKA/JOAはデフォルトでNone引数（後方互換）
- UKCは馬マスタのため特徴量には直接使用せず（将来の馬名/血統検索基盤）
