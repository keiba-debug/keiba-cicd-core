# v5.5 Bootstrap信頼区間 + 特徴量Pruning実験

> Session 44 (2026-02-22)
> 目的: VB ROI推定の信頼性を定量化し、特徴量pruningの効果を検証

---

## 背景

Session 43で「悩み2: バックテストROIの信頼性」を議論。月23件のgap>=5 VBでは統計的判断が困難。
Bootstrap信頼区間をexperiment.pyに実装し、全てのVB ROI評価にCIを付与する。

---

## 実装

### Bootstrap CI (`calc_vb_bootstrap_ci`)
- **レース単位リサンプリング**（馬単位では独立性の仮定が崩れる）
- 1000回Bootstrap、95% CI（2.5th - 97.5th percentile）
- VB均一買い（100円）のROI分布を推定

### bet_engine Bootstrap CI
- bet_engineプリセットのROIにもBootstrap CI追加
- レース単位でグループ化し、betとreturnをリサンプリング

### 特徴量Pruning (`--prune-bottom N`)
- 前回のexperiment結果からimportance下位N%の特徴量を自動除外
- Model A/Bそれぞれのimportance下位を独立に除外

## 分析・結論

### 1. Bootstrap CIが示す「不確実性の大きさ」

**最重要の発見**: win_only 95% CI = [60.2% - 170.4%]。幅が110pp以上ある。

- 366件のbet（≒半年分）では、ROIが60%（大損）から170%（大勝）まで幅がある
- 「ROI 119%」という点推定は**サンプルノイズに大きく左右される**
- **バージョン間の5-10ppの差は統計的にほぼ無意味**

### 2. Pruning 20%は「やりすぎ」

- Place VBは微改善（gap>=3-4で+1-2pp）だが、Win VBが大幅悪化（gap>=5: -27pp）
- WV AUC -0.0013は小さくないダメージ
- career_stageとcomment系を一緒に消したのが原因の可能性
- **dead features（importance=0）のみ削除がより安全**

### 3. 次のステップ

1. **dead features pruning**: importance=0の特徴量のみ除外（is_koukaku_age, is_koukaku_handicap等）
2. **bet数を増やす方向**: CIを狭めるには対象bet数を増やすしかない
   - gap>=4に条件を緩和 → bet数が倍増
   - margin閾値を緩和 → bet数増加
3. **時系列Cross-validation**: テスト期間を変えてROIの安定性を確認
4. **ベースラインのbet_engine CIも取得**: v5.4.1で再実行（次回）
