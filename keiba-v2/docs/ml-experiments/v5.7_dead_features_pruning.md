# v5.7 Dead Features Pruning

## 概要
Session 45（継続）: importance=0の特徴量のみを対象とした安全なpruning実験。
Session 44で「下位20%一括削除」が-27pp悪化（CI幅内でノイズの可能性あり）したため、
理論上モデル貢献ゼロの特徴量のみを削除して再学習し、影響を検証する。

## 背景
- Session 44: `--prune-bottom 20` → Win VB ROI -27pp悪化（CI=[60-170%]内）
- importance=0はLightGBMが一度もsplitに使用しなかった特徴量
- 理論上、除去してもモデル出力は変わらないはず
- ただし再学習時のfeature_fraction等の確率的要因で結果が変動する可能性あり

## 実験設定
- train: 2020-2023, val: 2024, test: 2025-2026
- ベースライン: v5.5（v5.4.1分類モデル + 回帰モデル、train=2020-2023）
- `--exclude-features` フラグを experiment.py に新規実装

## 判定: ロールバック

ROIが-18.3pp悪化（ルール: -5pp以上で中止）のためロールバック実施。
v5.5をliveモデルとして復元。

## 考察: なぜimportance=0の除去でROIが変動したか

### 1. 再学習の確率的ノイズ
LightGBMのfeature_fraction（=0.8）により、各ブースティングラウンドで使用する特徴量がランダムに選択される。114→111特徴量の変化は、このランダムサンプリングの挙動を変える。

### 2. Early Stoppingの連鎖的影響
- Model Wのiteration数が131→80と大幅に変動（-39%）
- これはWin modelのランキング精度に直接影響
- Win VB gapの計算に使われるため、ROIに波及

### 3. CI幅との関係
- ベースラインCI: [65.8-175.4%]（幅=109.6pp）
- -18.3ppの差はCI幅の17%に過ぎない
- 統計的には「両者に差がある」とは言えない

### 4. Session 44の-27ppと同じ構造
Session 44の20%pruning(-27pp)も、今回の0%pruning(-18pp)も、
本質的にはLightGBMの再学習ノイズである可能性が高い。
「特徴量を変更した→ROIが変わった」ではなく
「再学習した→ROIが変わった」が正確な因果関係。

## 重要な学び

1. **importance=0でも除去は安全ではない**: 再学習による確率的変動がROIに大きく影響
2. **AUC安定 ≠ ROI安定**: AUC差<0.002でもROI差が18ppに達しうる
3. **pruning実験の限界**: ROIのCI幅が100pp超の状態では、pruningの効果を検出不能
4. **再学習そのものがリスク**: 同じデータ・同じ設定でも再学習すればROIが±20pp変動する
5. **特徴量pruningは件数問題の解決後に再検討**: CI幅を許容レベル（例: ±15pp）に縮小してから

## 次のアクション
- ~~dead features pruning~~ → 中止（再学習ノイズに埋もれて効果判定不能）
- 現行v5.5モデルを維持（再学習しない）
- CI縮小（件数増加）を引き続き最優先課題とする
- 特徴量pruningはCI幅が十分に小さくなった時点で再検討
