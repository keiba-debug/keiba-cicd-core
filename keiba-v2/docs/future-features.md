# 今後の開発候補

## Web UI 改善

### レース詳細画面にJRDB由来データ統合
- IDM（独自指数）を馬情報に表示
- JRDB SED/KYI由来のデータ（前走IDM、総合指数等）
- レース画面で市場情報との比較が可能に

### 馬成績画面にJRDB由来データ統合
- 過去走ごとのIDM推移
- JRDB独自指標のトレンド表示
- 能力変化の可視化

### IDM比較画面の拡張
- **コース/距離フィルタ**: 同一コース条件のIDMだけを抽出して比較
- **ペースフィルタ**: Hペース/Mペース/Sペース別のIDM傾向
- **IDM比較 → 過去同一重賞比較**: 例えば弥生賞のIDM推移を過去5年分重ねて「今年のメンバーレベル」を把握
  - X軸: 各年のレース日、Y軸: IDM → 年ごとの線で比較
  - 同一レース名でdata3から過去開催を自動検索

### ラップ比較画面（IDM比較のアーキテクチャ転用）
- **同一コース過去レースのラップ重ね比較**: X軸=ハロン区間、Y軸=ラップタイム
- 過去の弥生賞ラップと今年の想定ペースを重ねてレースイメージを掴む
- 凡例ON/OFF + ハイライト連動はIDM比較と同一パターン
- JRDB SED通過順位データとの連携で馬別ラップも可能に

## 新シミュレーション

### 厳選3連単勝負シミュレーション
- 3連単を1点〜10点に絞って買い続けた場合の収支シミュレーション
- W/P/ARモデルの予測を組み合わせて3連単の軸・ヒモを選定
- ポイント:
  - 軸馬の選定方法（rank_w 1位? AR偏差値最大?）
  - ヒモ馬の選定方法（rank_p上位? EV順?）
  - フォーメーション設計（1着固定 or 1-2着固定）
  - backtest_cacheから3連単結果を取得する仕組みが必要
  - mykeibadbの3連単払戻データ連携

## 運用改善

### JRDB データダウンロード実行ボタン（Web UI）
- 管理画面からワンクリックでJRDBデータ（SED/KYI/KAA）を最新化
- ダウンロード進捗表示、完了後にindex再構築を自動トリガー
