## 位置づけ

MLロードマップとの役割分担：

```
MLロードマップ（既存）
└── 「どの馬が来るか」の確率を出す
    入力: レースデータ・調教データ・オッズ
    出力: 各馬のcalibrated確率・gap・margin・EV
    精度指標: AUC、ECE、ROI
    現在地: v5.5、CI縮小が最優先課題

馬券設計エンジンロードマップ（本書）
└── 「どの馬券をいくら買うか」を決める
    入力: MLが出した確率分布・オッズ・資金残高
    出力: 馬券種・買い目・金額配分
    精度指標: 期待値・的中率・Max DD
    現在地: 単勝/複勝のみ、手動判断
```

2つは**入力と出力の関係**であり、独立して進化させる。
MLの確率精度が土台であり、馬券設計エンジンはその上に乗る。

---

## ビジョン

**「同じ◎〇▲から、状況に応じた最適な馬券と金額を自動生成する」**

人間の熟練馬券師が行う以下の判断を、データドリブンで再現する：

- 単勝で絞るか、ワイドで広げるか
- 人気馬の頭を消すか残すか
- フォーメーションの組み方と点数配分
- 本線とプラスアルファの使い分け

---

## 設計原則

### 1. MLと馬券設計は分離する

馬券設計エンジンはMLモデルの出力（確率・スコア）を受け取るだけで、
モデル内部には干渉しない。MLの精度改善とは独立して進化できる。

### 2. 期待値が全ての判断基準

```
買い目の期待値 = Σ（的中確率 × 配当） - 投資額
期待値 > 0 の買い目のみを推奨する
```

「直感」や「パターン」ではなく期待値計算が買い目選択の根拠。

### 3. 試行回数の戦略を選択できる

```
戦略A（件数重視）: 複勝・ワイドで広く、DDを抑える
戦略B（絞り込み）: 単勝・三連単で深く、高ROIを狙う
```

どちらか一方に依存せず、両戦略を並立させることで脆弱性をなくす。

### 4. 金額は確信度に比例させる

均一買いではなく、期待値と確信度に応じてKelly基準で傾斜をつける。
ただしKelly過多のリスクを避けるため、上限キャップを設ける。

### 5. 人間の判断を残す

当日の外部条件（馬場・枠順・パドック）は人間が補正できる余地を設ける。
完全自動化より「自動提案 + 人間の最終判断」を当面の理想とする。

## フェーズ設計

### Phase 1: 基盤整備（現在〜2026年末）

**目標**: 単勝・複勝の買い目と金額をデータドリブンに決定する

```
実装項目:
├── Standard/Wide二軸プリセット（bet_engine v2）※実装中
├── ★スコアによるKelly傾斜（★3=通常、★1=半Kelly）
├── レースの荒れやすさスコア（荒れ確率0〜1）
│   └── 入力: 頭数・クラス・人気分布・ペース予測
└── 複勝戦略の基盤検証
    └── 危険馬×VB同一レース複勝ROIの確認
```

**成功指標**: Standard条件でMax DD<50%を6ヶ月維持

---

### Phase 2: 馬券種の自動選択（2027年目安）

**目標**: レース特性に応じて馬券種を自動推奨する

```
実装項目:
├── 荒れ確率に応じた馬券種マッピング
│   荒れ確率高い → ワイド・三連複推奨
│   荒れ確率低い → 単勝・馬連推奨
├── 複数馬スコアの相対差による馬券種決定
│   ◎と〇の差が大きい → 単勝 or 馬連
│   差が小さい → ワイド・三連複
└── EV<1.0の人気馬を自動で「頭消し」候補に
    → 〇の頭を消した◎▲軸の馬連・三連単
```

**成功指標**: 推奨馬券種のROIが手動選択を上回る

---

### Phase 3: フォーメーション自動生成（CI縮小後・2028年以降）

**目標**: 全馬の確率分布から最適な組み合わせと金額を自動計算する

```
実装項目:
├── 全馬のcalibrated確率から組み合わせEVを列挙
│   例: ◎▲→〇→◎▲（三連単）の期待値計算
├── 点数と期待値のトレードオフ最適化
│   点数増加 → 的中率上昇 vs 純利益減少
├── 本線とプラスアルファの自動分離
│   本線: EV最大の組み合わせに集中投資
│   プラスアルファ: 荒れ時の高配当を少額でカバー
└── 資金残高・目標ROIに応じたポートフォリオ配分
```

**成功指標**: フォーメーション買いのROIが単勝のみを上回る

## 現状の課題と未解決問題

| 課題 | 難易度 | 解決の方向 |
|---|---|---|
| 複勝ROI<100%問題 | 高 | 危険馬検出との組み合わせで突破口を探る |
| 三連単配当の事前予測 | 高 | 人気分布から期待配当を推定する近似モデル |
| 点数最適化の計算量 | 中 | 上位N頭に絞った枝刈りで対処 |
| 当日オッズ変動 | 中 | 締切直前オッズをリアルタイム取得して再計算 |
| Kelly過多リスク | 低 | 上限キャップ（例: 資金の5%/レース）で対処 |

## MLロードマップとの依存関係

```
MLロードマップの進捗      馬券設計エンジンへの影響
─────────────────────────────────────────────
CI縮小（±50pp）    → Phase 2の馬券種推奨の信頼性向上
CI縮小（±30pp）    → trainer_pattern_score実装→馬の確信度向上
CI縮小（±15pp）    → Phase 3フォーメーション計算の根拠が確立
当日外部条件入力    → 両フェーズ共通で精度向上
```

馬券設計エンジンの精度はMLの確率精度に依存する。
MLが改善されるほど、馬券設計エンジンの判断根拠が強化される。

## 成功の定義

```
短期（Phase 1完了時）:
・単勝のMax DD < 50%を安定維持
・複勝戦略の有効性が統計的に確認できる

中期（Phase 2完了時）:
・馬券種の自動推奨が手動判断を上回るROI
・試行回数重視・絞り込みの両戦略が並立

長期（Phase 3完了時）:
・フォーメーション買いの期待値が最大化される
・「買い目と金額」の判断が完全に自動化される
・人間は当日条件の補正と最終承認のみ担当
```
