## 1. プロジェクト概要

### 目的

```
KeibaCICDの予測モデル・各種指標を使って
WIN5を現実的に的中させる買い方を探索する

「的中するために最低何点必要か」
「どんな日を狙い・どんな日を捨てるか」
「何円投資すれば現実的か」
「そもそも的中は現実的に可能か」

を過去データのシミュレーションで明らかにする
```

### 重要な前提：WIN5はEVではなくカバレッジの問題

```
通常のVB戦略:
  同じ馬を繰り返し買う → EV（期待値）が重要

WIN5:
  的中か外れかの二択
  → 「勝つ馬を何点でカバーできるか」だけが問題
  → EVは使わない

最適化の本質:
  最小点数で5レース全ての勝ち馬をカバーする
  = 点数 × カバレッジの最適バランス
```

### あきらめ基準（プロジェクトの結論として重要）

```
以下の結果が出た場合はWIN5を断念する

断念基準A: 的中に平均500点以上必要
  → 50,000円/週 × 52週 = 年間260万円の投資
  → 非現実的

断念基準B: 「週を選んでも」100点以下での的中が
  過去3年で0〜1回
  → 狙い目週の選別に意味がない

断念基準C: 狙い目スコアの高い週と低い週で
  必要点数に有意差がない
  → 週の選別戦略が機能しない

→ 「あきらめる」という結論も
  重要な意思決定の成果
```

### WIN5の基本スペック

```
対象: 毎週日曜 指定5レース（原則G1〜OP）
最低購入: 100円/点
払戻: 売上の約72%（控除率28%）
理論的難易度: 平均15頭×5R = 1/759,375

KeibaCICDのTop1勝率: 約32.7%
→ Top1×5R全的中確率: 0.327^5 ≈ 0.39%
→ 単純計算で256回に1回（約5年に1回）

Top2まで選択時の各R的中率: TBD（要集計）
Top3まで選択時の各R的中率: TBD（要集計）
```

## 2. 使用する指標

```
WIN5で使う指標:
  ✅ pred_proba_wv（勝利確率）← カバー馬の選別（★最重要）
  ✅ ARd（レース内偏差値）   ← 格下除外
  ✅ 危険な人気馬フラグ       ← 消し条件
  ✅ 出走頭数                ← 点数の見積もり
  ✅ レースグレード           ← G1 = 狙い目（v_Top1: 47.8%）
  ✅ 馬場状態                ← 重・不良 = 参加不可
  △ 本紙印                  ← 消し専用（ARd<45&無印=1.6%）

WIN5で使わない指標:
  ❌ EV（期待値）             ← 長期反復投資の指標
  ❌ Gap                     ← 同上
  ❌ VBフラグ                 ← 同上
  ❌ 本紙印（選択用）         ← 無印勝率29.7% > ◎勝率24.1%
  ❌ BR                      ← WIN5では検証未実施
```

## 3. 分析の切り口

### 切り口①：日程・開催の選別

**狙うべき日**

```
条件A: 一点突破できるレースが多い日
  → ARd1位 >= 65 かつ 2位との差 >= 10
  → 5R中3R以上で該当

条件B: 少頭数レースが多い日
  → 12頭以下が多い = 組み合わせ点数が少ない
  → 10頭×5R = 100,000点
  → 8頭×5R  =  32,768点
  → 頭数削減の効果は大きい

条件C: 危険な人気馬がいる日
  → 「消せる馬」がいる = 実質的に頭数が減る
  → 点数削減につながる

条件D: 実力差が明確な日
  → 古馬G1（最も実力差がハッキリしやすい）
  → 芝レースが多い（ダートより予測精度が高い）
```

**捨てるべき日**

```
条件X: ランダム性が高いレースが多い日
  → ダート短距離（砂の跳ね方でランダム性大）
  → 3歳牝馬限定戦（実力差が小さい）
  → 新馬・未勝利が対象に含まれる（実績データ少）

条件Y: 多頭数レースが集中する日
  → 18頭×18頭が2レース以上ある週
  → 点数が爆発して現実的でない

条件Z: 重・不良馬場が予想される日
  → 予測精度が低下する
  → 荒れやすくカバレッジが困難
```

---

### 切り口②：レース別の買い方分類

**タイプA: 一点突破レース**

```
定義:
  ARd1位 >= 65 かつ 2位との ARd差 >= 10
  かつ pred_proba_wv >= 40%

買い方:
  1頭のみ（1点）

例:
  「圧倒的1強」レース → 迷わず1頭で固定
```

**タイプB: 信頼軸レース**

```
定義:
  ARd1位 >= 60 かつ 2位との差 >= 6

買い方:
  1〜2頭（軸固定）

例:
  「強いが100%ではない1強」
  → Top1固定でTop2を押さえる
```

**タイプC: 混戦レース**

```
定義:
  ARd上位3頭の差が10以内

買い方:
  ARd >= 50 の馬を全買い（3〜5頭）
  → 格下（ARd<45）だけ除外して広くカバー

例:
  「誰が勝つか分からないハンデ戦」
```

**タイプD: 危険人気馬消しレース**

```
定義:
  1〜3番人気に消し条件該当馬がいる
  （odds<=8 & ARd<50 & V%<15%）

買い方:
  該当馬を除外した上でタイプA/B/Cを適用
  → 実質的な頭数削減 = 点数削減

効果:
  18頭立てから1〜2頭消せれば
  組み合わせが約10〜20%減少
```

---

### 切り口③：投資額別の参加基準

**〜1,000円（〜10点）**

```
参加条件: 全5RがタイプAのみ
現実性: 極めてレア（年0〜2回）
→ 「奇跡の週」専用
```

**〜5,000円（〜50点）**

```
参加条件: タイプA×3 + タイプB×2
現実性: 年数回
→ バックテストで存在確認が必要
```

**〜10,000円（〜100点）**

```
参加条件: タイプA×2 + タイプB×2 + タイプC×1
現実性: 月1〜2回程度
→ メインの検証ターゲット
```

**〜30,000円（〜300点）**

```
参加条件: 狙い目スコア60点以上の週
現実性: 毎週に近い
→ 上限ラインの確認用
```

**500点以上**

```
→ あきらめ基準Aに該当
→ この点数が必要な週は参加しない
```

## 5. 仮説

### 仮説①（楽観）：狙い目週なら100点以内が年数回ある

```
タイプA×3以上の週が年5〜10回存在する
→ この週は100点以内で的中できた可能性がある
→ WIN5に挑戦する価値がある
```

### 仮説②（中立）：最低でも数十〜百点は必要

```
現実的な最小点数の試算:
  A×A×A×B×C = 1×1×1×2×4 = 8点  ← 理想的な週
  A×B×B×C×C = 1×2×2×4×4 = 64点 ← 現実的な週
  B×B×C×C×C = 2×2×4×4×4 = 256点 ← 混戦週

→ 週のタイプ構成で大きく変わる
```

### 仮説③（悲観）：構造的に割に合わない

```
的中に平均500点必要（50,000円）
払戻の期待値 = 50,000 × 0.72 = 36,000円
→ 毎週買い続ければ必ず損をする
→ 断念基準Aに該当 → プロジェクト終了
```

### 仮説④：危険人気馬の消しがWIN5でも有効

```
消し条件（odds<=8 & ARd<50 & V%<15%）の馬が
WIN5対象レースで実際に負けた割合が
通常レース同様に高ければ有効

→ WIN5対象レースに限定した消し精度を別途集計
```

## 6. 実装ステップ

```
Step 1: データ準備
  □ 過去のWIN5対象レースを特定（何週分あるか）
  □ KeibaCICD予測データとの紐付け
  □ 実際の払戻金データを収集
  □ 各週「的中に最低必要だった点数」を算出

Step 2: 基本シミュレーター実装
  □ Top1/2/3での的中回数と最小点数を集計
  □ 断念基準A〜Cの判定を出力
  □ 「1万円以内で的中できた事例」を可視化

Step 3: レースタイプ分類の実装
  □ タイプA/B/C/D の自動判定
  □ 週の狙い目スコア計算

Step 4: 戦略比較レポート
  □ 全戦略の点数・的中率・払戻比較
  □ 断念 or 継続の結論を出す

Step 5:（継続の場合のみ）定期運用組み込み
  □ 毎週「今週のWIN5狙い目スコア」を自動計算
  □ 推奨買い目を自動生成
```
