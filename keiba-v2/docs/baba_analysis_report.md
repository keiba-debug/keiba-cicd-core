## 10. 条件不利凡走の割引 → 条件好転時の狙い目

### 概念: 「馬場不利フラグ」

過去走で以下の条件に該当する凡走は「条件不利」として割り引くべき:

| パターン | 不利条件 | 割引対象 |
|---------|---------|---------|
| **差し馬 @ 福島芝序盤** | 差し有利度 -19.1pt | 内先行有利で差せなかった |
| **差し馬 @ 短距離ダート湿潤** | 差し有利度 -24.6pt | 物理的に差しが効かない |
| **外枠 @ 中山芝** | 内外差 +6.0pt(標準) | コースレイアウト上の不利 |
| **追込馬 @ 小倉ダート湿潤** | 追込top3 3.8% | 巻返し率8.5%で追い込み困難 |

### 逆に「条件好転」で狙うべき切替パターン

| 前走の不利 | 今回の好転条件 | 期待される効果 |
|-----------|-------------|-------------|
| 福島芝序盤で差し凡走 | → 東京芝中盤以降 | 差し有利度 -19.1→-12.5pt (+6.6pt) |
| 中山芝外枠で凡走 | → 新潟芝後半 | 外枠 top3 20.8→24.1% |
| 阪神芝硬で差し凡走 | → 阪神芝後半+柔 | 外差し 19.4→**40.0%** |
| 短距離ダート湿潤で差し凡走 | → 中長距離ダート乾燥 | 差し有利度 -24.6→-13.8pt |

### 実装案（将来課題）

1. **前走の条件不利フラグ**: 過去走の会場×開催日×クッション/含水率から、その馬の脚質にとって不利だったかを判定
2. **excuse_flag拡張**: 現行の`comment_stable_excuse_flag`に加えて、馬場由来のexcuseを追加
3. **条件好転検出**: 今回のレース条件が、前走の不利条件と逆のパターンかを判定
4. **AR補正候補**: 前走不利凡走の着順を「割り引いて」ARd計算に反映

→ これらは分析レポートとして人が判断する段階から始め、ML特徴量化は段階的に検討

**超短×標準クッション = 内有利度+4.6ptが最大**。柔だと+1.8ptに低下。

**ダート: 1角距離×含水率→内有利度**

| 1角距離 | 乾燥(<8%) | 標準(8-12%) | 湿潤(>=12%) |

| 中 | -4.8pt | -4.1pt | -2.6pt |
| **長** | **-7.3pt** ★ | **-8.0pt** ★ | -4.5pt |

**長距離×乾燥/標準 = 外枠有利-7〜-8ptが最大**。湿潤だと-4.5ptに緩和。
