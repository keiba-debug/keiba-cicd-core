**不採用理由**: 曖昧な語の追加はノイズになりROI低下。精度の高い少数語が最適。

## 学び

### 1. Feature importance低くてもVB ROI改善可能
コメント特徴量はModel B #72-91/91（最下位グループ）だが、Place VB gap>=5は+4.9pp改善。
特徴量の直接的な重要度とVB ROIへの寄与は別物。

### 2. NLP辞書は少数精鋭が有効
辞書拡充版(v5.3b)よりシンプル版(v5.3)がVB ROI高い。
曖昧語（「まずまず」「それなり」等）はポジティブにもネガティブにも解釈でき、ノイズになる。

### 3. Win VB ROI低下の代償
Place VB gap>=5 +4.9ppの裏で、Win VB gap>=5 -8.4pp。
コメント情報が複勝の穴馬検出に効くが、単勝精度には逆効果の可能性。
