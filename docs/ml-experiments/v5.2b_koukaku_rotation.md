# ML v5.2b: 降格ローテ + レースレベル特徴量（+11特徴量）

**日付**: 2026-02-20
**種別**: 特徴量エンジニアリング + データ基盤
**ベース**: v5.1（v5.2はロールバック済み）

---

## 概要

降格ローテ理論を11個のML特徴量に変換。前走よりグレード/会場ランクが下がる
ローテーション（降格ローテ）を検出し、パターン別にフラグ化。

同時に `race_level_index.json` を新規生成し、レースレベル判定（H/M/L）の
データ基盤を構築。horse_history_cacheにgrade/venue_name等を追加。

## 新特徴量（11個）

### MARKET特徴量（3個）- Model A専用
出馬表から自明に判定可能 = オッズに反映済み。

| 特徴量 | 説明 | Model A |
|--------|------|---------|
| `prev_grade_level` | 前走のグレードレベル(1-10) | 1,174 (#42/98) |
| `venue_rank_diff` | 会場ランク差(A=5...E=1) | 947 (#48/98) |
| `grade_level_diff` | グレードレベル差 | 671 (#63/98) |

### VALUE特徴量（8個）- Model A+B
条件の組み合わせは市場に見えにくい。

| 特徴量 | 説明 | Model A | Model B |
|--------|------|---------|---------|
| `koukaku_rote_count` | 降格パターン合計数 | 210 (#83) | 1,335 (#69/82) |
| `is_koukaku_venue` | 会場ランク下降 | 137 (#86) | 761 (#73) |
| `is_koukaku_female` | 牝馬限定 -> 混合戦 | 230 (#82) | 670 (#74) |
| `is_koukaku_distance` | 距離短縮(200m+) | 18 (#95) | 356 (#75) |
| `is_koukaku_season` | 秋 -> 冬春(1-3月) | 44 (#91) | 292 (#76) |
| `is_koukaku_turf_to_dirt` | 芝 -> ダート転向 | 64 (#90) | 279 (#77) |

## 分析

### ポジティブ
- **Value系モデル(B/WV)のAUCが改善**: 降格ローテは市場に見えにくい信号 -> VB哲学と合致
- **Model WV AUC +0.0013は有意**: 全バージョン中最高の0.7873
- **Model B AUC 0.7843**: 全バージョン中最高

### ネガティブ
- **VB ROIは低下**: gap>=5で-10~-15pt
- **VALUE特徴量の重要度が全体的に低い**: Model Bでもkoukaku_rote_count(#69/82)が最高
- **Dead features 2個**: is_koukaku_age/is_koukaku_handicap
- **ECE微悪化**: 特徴量数増加の副作用

### 次のステップ
降格ローテ特徴量の取捨選択が必要:
- **残す候補**: koukaku_rote_count + is_koukaku_venue + is_koukaku_female (上位3)
- **除外候補**: is_koukaku_distance/season/turf_to_dirt (低重要度)
- **削除**: is_koukaku_age/is_koukaku_handicap (dead)
- **TODO**: is_koukaku_age実装にはrace_class情報が必要

## 学び

### 1. AUC改善とROI低下の共存
Model B/WV AUCが改善してもROIが下がるケースがある。
特徴量追加による小さなAUC改善は、VB gap計算のノイズを増幅する可能性。

### 2. MARKET/VALUE分類は重要
prev_grade_level(#42)はModel Aでそこそこ有用。出馬表から自明な情報を
MARKETとして適切に分離した設計が妥当。

### 3. 降格ローテは理論的に正しいが特徴量化が難しい
競馬理論としての降格ローテは有効（人間の予想では活用されている）だが、
個別パターンのフラグ化では重要度が低い。複合条件や相互作用の検出が必要かもしれない。
