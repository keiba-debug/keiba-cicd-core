# ML v5.1: 33ラップ + レース傾向適性特徴量（+5特徴量）

**日付**: 2026-02-20
**種別**: 特徴量エンジニアリング + 分類基盤

---

## 概要

新規の `race_classifier.py` でレースの7分類v2と33ラップ（前半3F/後半3Fの差）を算出。
これを馬の過去走と組み合わせ、瞬発力/持続力の適性と傾向別好走率を特徴量化。

全20,415レースJSONを再ビルドし、race_type_standards v5.0を更新。

## 新特徴量（5個）

| 特徴量 | 説明 | Model A | Model B | Model WV |
|--------|------|---------|---------|----------|
| `prev_race_lap33` | 前走の33ラップ（瞬発/持続度合い） | 3,482 (#10) | **9,830** | **22,270** |
| `avg_lap33_last3` | 直近3走の平均33ラップ | 3,261 (#11) | **8,205** | **21,026** |
| `trend_versatility` | 傾向別top3率の標準偏差 | 1,719 | 5,756 | 12,643 |
| `worst_trend_top3_rate` | 最も苦手な傾向でのtop3率 | 1,113 | 3,321 | 8,312 |
| `best_trend_top3_rate` | 最も得意な傾向でのtop3率 | 1,032 | 3,481 | 7,013 |

### 33ラップとは
```
lap33 = (後半3F - 前半3F)
負の値 = 後半が速い = 瞬発力レース（上がり勝負）
正の値 = 前半が速い = 持続力レース（消耗戦）
```

### MARKET_FEATURES判定
- **結論: 5特徴量全てModel A+B両方に残す**
- 33ラップはコース構造由来の物理特性 -> 市場が完全に織り込んでいるとは考えにくい
- Model WVでの突出した重要度がValue特徴量としての有効性を裏付け

## 学び

### 1. ECE改善が最大の成果
全4モデルでキャリブレーション向上。33ラップが確率推定の精度を底上げ。
AUCは横ばいだが、確率の絶対値がより正確になった。

### 2. Win モデルに特に有効
Model W AUC +0.0007、Model WV AUC +0.0009。勝ち馬予測に33ラップの
物理情報が効く。瞬発力レースか消耗戦かで勝ちパターンが変わる。

### 3. Model WVでの重要度が突出
prev_race_lap33=22,270、avg_lap33_last3=21,026。市場除外モデルで最も輝く特徴量。
オッズには反映されにくいコース物理特性の情報。

### 4. Place VB 単勝ROI大幅改善
gap>=4で+12.9pt、gap>=5で+12.3pt。VB候補内の勝ち馬特定精度が向上。
Win/Placeデュアル体系の効果が33ラップで増幅された形。

### 5. Place VB 複勝ROIは微減
gap>=5で-4.9pt。サンプルサイズ(564件)を考慮すると統計的ノイズの可能性。
