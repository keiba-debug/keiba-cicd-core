# Value Bet 画面改善提案（新方式対応）

## 背景

Value Bet画面は**実運用時の意思決定画面**。
ML Reportが「分析・チューニング」ツールなのに対し、
Value Betは「今日どの馬にいくら賭けるか」を決めるツール。

bet-engine.py 導入で以下が変わる:
- 推奨ロジックがフロントのuseMemoからPython側JSONに移行
- Win/Place独立判断（旧: 芝=単勝, ダート=複勝 の固定ルール廃止）
- marginフィルタ追加（回帰モデル予測値）
- Win側は確率ベースEV不使用（gap比例sizing）
- Place側はcalibrated EV + Kelly

現画面は旧ロジック前提のUI設計なので、新方式に合わせた改善が必要。

## 画面別改善案

### 1. サマリータブ

#### 現状
```
VB成績サマリー  結果反映済
区分     VB数  単的中  単勝ROI      複的中  複勝ROI
全体      60   2/60   55.2%(-2,690) 5/60   51.5%(-2,910)
芝        32   0/32    0.0%(-3,200) 1/32   40.3%(-1,910)
ダート     25   2/25  132.4%(+810)  4/25   72.0%(-700)
```

#### 問題点
- 芝/ダートの分離表示は「芝=単勝、ダート=複勝」の旧戦略を前提にしている
- bet-engine推奨の損益が表示されていない（VB全体の均一買いROIのみ）
- marginフィルタの効果が見えない

#### 改善案

**1-1. bet-engine推奨損益を最上部に表示**

サマリーの最重要情報は「bet-engineに従っていたらいくら儲かったか」。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
bet-engine 成績 (standard)          結果反映済
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
推奨数: 12件  投資総額: ¥4,800  回収: ¥6,240
  単勝: 5件 ¥1,800 → ¥3,040 (168.9%) ✅
  複勝: 9件 ¥3,000 → ¥3,200 (106.7%) ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
総合ROI: 130.0%  損益: +¥1,440
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[standard ✓] [conservative] [aggressive]  ← プリセット切替
```

**1-2. VB成績サマリーの分類軸を変更**

芝/ダート → bet推奨/非推奨 に変更:

```
VB成績サマリー（均一買い参考値）
区分           VB数  単的中  単勝ROI   複的中  複勝ROI
全体            60   2/60   55.2%     5/60   51.5%
bet推奨(12件)   12   2/12  168.9%     4/12  106.7%  ← bet-engine通過分
bet非推奨       48   0/48    0.0%     1/48   33.3%  ← フィルタ除外分
```

これにより「bet-engineのフィルタがどれだけノイズを除去したか」が一目瞭然。
bet推奨のVBだけ買っていれば黒字、非推奨を買うと大赤字、という構図。

**1-3. 芝/ダート分離は折りたたみで残す**

参考値として残すが、デフォルトでは折りたたみ:

```
▶ 馬場別内訳（参考）
  芝:    32件  単0.0%   複40.3%
  ダート: 25件  単132.4% 複72.0%
```

### 2. 推奨買い目タブ

#### 現状
```
推奨買い目 (50件)  Stage 1
[標準] [複勝厳選] [攻め] [カスタム▼]  自動 / Gap≥3 / Kelly10%

¥16,400    18件      36件     1.27      ¥20,801    6R
投資総額   単勝¥2,600 複勝¥13,800 加重平均EV  期待回収額  危険人気馬

場  R  馬番 馬名    推奨  単EV 複EV Gap Kelly 金額▼ オッズ WV% 頭% 危険馬 着順 配当
```

#### 問題点
- 単EV / 複EV は empirical rates ベースの旧計算（Model WV ECE=0.12で不正確）
- Kelly% は Win/Place 共通の一本値（新方式では Place のみ Kelly）
- プリセットが旧名称（標準/複勝厳選/攻め）
- 「Stage 1」表記の意味が不明
- marginが表示されていない
- 「期待回収額」は empirical EV × 金額の合計で、信頼できない

#### 改善案

**2-1. カラム構成の変更**

旧:
```
推奨 | 単EV | 複EV | Gap | Kelly | 金額 | オッズ | WV% | 頭%
```

新:
```
推奨 | Gap | Margin | 金額(Win/Place) | オッズ | 複EV | Kelly | 強度 | 理由
```

変更ポイント:
- `単EV` 削除 → Win側はEV計算しない（WV ECE不適格のため）
- `Margin` 追加 → 回帰モデル予測値、フィルタの根拠
- `金額` → Win/Place分離表示（新方式では独立計算）
- `Kelly` → Place側のみ表示（Win側はgap比例）
- `強度` 追加 → 'strong'(gap≥5) / 'normal'
- `理由` 追加 → 券種選択理由の1行サマリー

```
場  R  馬番 馬名       推奨   Gap  Margin  Win額  Place額  オッズ  複EV  Kelly  強度     理由
東京 2   3  ゴールドエンバー  単複   +5   0.82s   ¥200   ¥400   105.1  1.48  1.2%  strong  gap5+低margin
東京 3  12  エタンセル      単複   +5   0.95s   ¥200   ¥300    56.4  1.35  0.8%  strong  gap5+EV1.35
小倉 11  17  タマモブラックタイ 単勝   +5   1.15s   ¥200    -     19.1   -     -    strong  gap5,margin>1.0
京都 2  12  ショルタメンテ    複勝   +6   0.65s    -     ¥500     -    1.52  2.1%  strong  EV1.52,低margin
東京 4   3  ロンギングフォーユー 単勝   +5   1.18s   ¥200    -     10.4   -     -    strong  gap5
```

**2-2. プリセット名称の更新**

旧: `[標準] [複勝厳選] [攻め] [カスタム▼]`
新: `[standard] [conservative] [aggressive] [カスタム▼]`

または日本語:
`[標準] [堅実] [攻め] [カスタム▼]`

各プリセットの説明ツールチップ:
```
標準:      gap≥4, margin≤1.2(Win)/1.0(Place), EV≥1.0
堅実:      gap≥5, margin≤1.0(Win)/0.8(Place), EV≥1.2
攻め:      gap≥3, margin≤1.5(Win)/1.2(Place), EV≥0.9
```

**2-3. サマリーカードの変更**

旧:
```
¥16,400    18件      36件       1.27        ¥20,801      6R
投資総額   単勝¥2,600 複勝¥13,800 加重平均EV   期待回収額   危険人気馬
```

新:
```
¥4,800     5件        9件       130.0%     +¥1,440      3R
投資総額   単勝¥1,800  複勝¥3,000 実績ROI    実績損益     危険レース
```

変更ポイント:
- 「加重平均EV」→ 削除（Win側EVは使わないため、混合平均EVは無意味）
- 「期待回収額」→ 「実績損益」（結果反映済みの場合）or 未確定時は非表示
- 件数が大幅に減る（50件→12件程度、marginフィルタで絞り込み）
- 結果未反映時:

```
¥4,800     5件        9件        -          -           3R
投資総額   単勝¥1,800  複勝¥3,000  結果待ち    結果待ち    危険レース
```

**2-4. Stage表記の廃止**

「Stage 1」は旧実装の名残。新方式ではプリセット名のみ表示:

旧: `推奨買い目 (50件)  Stage 1`
新: `推奨買い目 (12件)  standard`

### 3. VB候補タブ

#### 現状
```
場 R 馬場 馬番 馬名  推奨 本紙 VR 人気 オッズ Gap▼ W-Gap 単EV 複EV 頭% V% WV% 調 談 短評 着順 単払 複払
```

#### 問題点
- `Margin` 列がない（新方式の核心フィルタ）
- `単EV` は不正確（WV ECE=0.12）
- VB候補がbet-engineで推奨/非推奨のどちらかわからない
- `W-Gap`（WVモデル版gap）と `Gap`（Vモデル版gap）の使い分けが不明確

#### 改善案

**3-1. Margin列の追加**

GapとW-Gapの間に挿入:

```
場 R 馬場 馬番 馬名  推奨 本紙 VR 人気 オッズ Gap▼ Margin W-Gap 複EV 頭% V% WV% 調 談 短評 着順 単払 複払
```

Margin列のセル色:
- ≤0.8s: 緑背景（有望帯）
- 0.8-1.2s: 黄背景（標準フィルタ通過）
- >1.2s: 赤文字（標準設定で単勝除外）
- >1.5s: 赤背景（aggressive設定でも除外）

**3-2. 単EV列の扱い**

2つの選択肢:

案A: 単EV列を削除（WV ECEが改善されるまで）
- pros: 不正確な情報を表示しない
- cons: 情報量が減る

案B: 単EV列を残すが、参考値表記に変更
- `単EV` → `単EV*` (アスタリスク)
- ツールチップ: "WVモデルのcalibration精度が低いため参考値"
- 文字色をグレーに

**推奨: 案B**（将来WV calibration改善時に復活させやすい）

**3-3. bet判定アイコンの追加**

推奨列の表示を拡張:

```
現行: [単複] [単勝] [複勝] （bet-engine推奨の券種のみ表示）
追加:  -     （推奨なし = bet-engineフィルタ除外）
```

除外理由をツールチップで表示:
```
- margin超過 (1.45s > 1.2s)
- gap不足 (3 < 4)
- Place EV不足 (0.89 < 1.0)
- 同一レース単勝制約で除外
```

**3-4. Marginフィルタの追加**

既存フィルタバーに追加:

```
場所: [全て] 東京 京都 小倉
R: [全て] 1 2 3 ...
馬場: [全て] 芝 ダート
Gap: ≥3 ≥4 ≥5
Margin: [全て] ≤0.8 ≤1.0 ≤1.2    ← 新規追加
EV: [全て] ≥0.8 ≥1.0 ≥1.2
□ bet推奨のみ                     ← 新規追加
```

「bet推奨のみ」チェックボックスで、bet-engine推奨の馬のみ表示。

### 5. レースカードタブ

#### 現状
```
1R  dirt1300m  16頭                          VB 2頭  詳細→

番▲ 馬名          紙 A順 V順 人 オッズ Gap 単EV A%  V%  WV% Rate 調 談 短評     着順 単払 複払
1  ビーエムキンチャン VB 複勝  - 12  8  12 193.5 +4  4.37 0.7 2.4  2.3 49.5 → 0 ダートで着順上げ 13
```

#### 問題点
- Margin列がない
- 単EVは不正確
- VBバッジの横に券種（複勝）が表示されているが、bet-engine推奨と一致するかわからない
- Rate列の意味が不明（V%とWV%の加重平均？）

#### 改善案

**5-1. Margin列の追加**

Gap列の隣に挿入。レースカードは全馬表示なので、
VBでない馬のmarginも見えることで「この馬はなぜVBでないか」がわかる。

```
番 馬名       紙 A順 V順 人 オッズ Gap Margin 単EV* A%  V%  WV% 調 談 短評    着順 単払 複払
1  ビーエムキンチャン  -  12  8  12 193.5 +4  1.85s  4.37 0.7 2.4 2.3  → 0  ダートで  13
```

margin=1.85sなら「gapは+4あるがmarginが大きすぎてbet-engine除外」とわかる。

**5-2. VBバッジの改善**

現行: `VB 複勝` （VBであること + 旧ロジックの券種）
新方式: bet-engine推奨の有無と券種を分離表示

```
VB候補 & bet推奨あり:  [VB] [単複 ¥200/¥400]
VB候補 & bet推奨なし:  [VB] [除外: margin>1.2]
VB候補でない:          (バッジなし)
```

色分け:
- `[単複]` `[単勝]` → 青背景
- `[複勝]` → 緑背景
- `[除外: ...]` → グレー背景

**5-3. bet-engine推奨行のハイライト**

bet-engine推奨の馬の行を薄い青背景でハイライト。
レースカードをスクロールする際に、推奨馬がすぐ見つかる。

**5-4. レースヘッダーにbet-engine情報追加**

```
現行: 1R  dirt1300m  16頭              VB 2頭  詳細→
新:   1R  dirt1300m  16頭  bet: 単複1件  VB 2頭  詳細→
```

bet推奨がないレースは:
```
3R  turf1600m  14頭  bet: なし   VB 0頭  詳細→
```

## 技術的変更点

### predict.py 出力JSONの変更

```json
{
  "date": "2026-02-22",
  "races": [ ... ],
  "recommendations": {
    "standard": [
      {
        "race_id": "202602221105",
        "umaban": 7,
        "horse_name": "ゴールドエンバー",
        "bet_type": "単複",
        "strength": "strong",
        "bet_amount_win": 200,
        "bet_amount_place": 400,
        "gap": 5,
        "predicted_margin": 0.82,
        "win_odds": 105.1,
        "place_odds_min": 3.2,
        "place_ev": 1.48,
        "kelly_place": 0.012,
        "win_units": 2,
        "exclusion_reason": null
      }
    ],
    "conservative": [ ... ],
    "aggressive": [ ... ]
  },
  "vb_exclusions": [
    {
      "race_id": "202602221105",
      "umaban": 10,
      "horse_name": "オソルノ",
      "gap": 8,
      "predicted_margin": 2.15,
      "exclusion_reason": "margin超過 (2.15s > 1.2s)"
    }

}
```

`vb_exclusions`: VB候補(gap≥3)だがbet-engine推奨から除外された馬のリスト。
除外理由を保持することで、VB候補タブで「なぜ推奨されなかったか」を表示可能。

### フロントエンド変更

```typescript
// predictions-content.tsx

// 旧: 100行のuseMemoでbet計算
// 新: JSONから読み込み + budget rescale のみ

const betRecs = useMemo(() => {
  const base = data.recommendations?.[preset] ?? [];
  return rescaleBudget(base, dailyBudget);
}, [data.recommendations, preset, dailyBudget]);

// VB候補にbet判定を付与
const vbWithBetStatus = useMemo(() => {
  const recMap = new Map(betRecs.map(r => [`${r.race_id}-${r.umaban}`, r]));
  const exclMap = new Map((data.vb_exclusions ?? []).map(e => [`${e.race_id}-${e.umaban}`, e]));

  return vbCandidates.map(vb => {
    const key = `${vb.raceId}-${vb.umaban}`;
    const rec = recMap.get(key);
    const excl = exclMap.get(key);
    return {
      ...vb,
      betRecommendation: rec ?? null,
      exclusionReason: excl?.exclusion_reason ?? null,
    };
  });
}, [vbCandidates, betRecs, data.vb_exclusions]);
```

### 削除対象

| 対象 | 理由 |
|------|------|
| `BACKTEST_EMPIRICAL` テーブル | calibrated確率に置換 |
| `getEmpiricalRates()` | 同上 |
| `getBuyRecommendation()` useMemo | JSON読み込みに置換 |
| Stage 1/2 の概念 | プリセット名に統一 |
| 芝=単勝/ダート=複勝 の固定ルール | Win/Place独立判断に移行 |
| 「加重平均EV」サマリーカード | Win EVが不正確のため意味をなさない |
