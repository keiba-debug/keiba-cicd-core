# ML v5.4: ベイズ平滑化レート + career_stage（+6特徴量）

**日付**: 2026-02-21 (Session 36)
**種別**: 特徴量エンジニアリング + SHAP診断
**ベース**: v5.3

---

## 概要

SHAP分析で「サマーゲール事件」（新馬戦快勝1戦1勝の馬がModel Bで最下位評価）の
原因を特定。LightGBMが `win_rate_all=1.0`（1戦1勝）を小標本として大きくペナルティ
していた。ベイズ平滑化レートで解決。

**重要な発見**: 平滑化レートをModel Bに入れるとA-B乖離が縮小してVB ROI低下。
**MARKET分類**にすることでModel Aの精度を上げつつ、VB効果を最大化。

## サマーゲール事件

- 東京2R ①サマーゲール: 新馬戦快勝、キング騎手、AI指数264.4A
- Model V順位=16（最下位）、Gap=-14で「危険な人気馬」判定
- **SHAP分析**: `win_rate_all=1.0` → SHAP寄与 **-0.1856**（Model B最大マイナス）
- `debug_features.py` 新規作成: `pred_contrib=True` でSHAP値を抽出する診断ツール

## 新特徴量（6個）

### MARKET特徴量（5個）— Model A専用
平滑化レートは市場情報を反映する（成績→オッズに直結）ためMARKET分類。

| 特徴量 | パラメータ | 説明 |
|--------|-----------|------|
| `win_rate_smoothed` | α=1.0, β=12.0 (prior≈7.7%) | ベイズ平滑化勝率 |
| `top3_rate_smoothed` | α=2.5, β=7.5 (prior=25%) | ベイズ平滑化複勝率 |
| `venue_top3_rate_smoothed` | 同上 | 場所別平滑化複勝率 |
| `track_type_top3_rate_smoothed` | 同上 | 馬場別平滑化複勝率 |
| `distance_fitness_smoothed` | 同上 | 距離適性平滑化 |

### VALUE特徴量（1個）— Model A+B
| 特徴量 | 説明 |
|--------|------|
| `career_stage` | キャリア段階 (0=debut,1=2戦,2=3-5,3=6-10,4=11+) |

### ベイズ平滑化の式
```
smoothed_rate = (n * raw_rate + α) / (n + α + β)
```
- n=0 → prior (α/(α+β))
- n大 → raw_rateに収束

## MARKET分類の根拠

## 学び

### 1. ベイズ平滑化 = MARKET
Model Aの精度向上に使いつつ、Model Bには入れない → A-B乖離拡大 → VB ROI最大化。
「精度向上がROI向上とは限らない」の典型例。

### 2. career_stage = VALUE
少数走馬の構造情報。Model Bに「2戦目」「デビュー」の文脈を与えることで
サマーゲール事件のような少数走馬の誤判定を軽減。

### 3. SHAP診断ツールの威力
`pred_contrib=True`で個別馬の特徴量貢献をリアルタイム分析可能。
原因不明の異常ランキングを即座に解明できる。
