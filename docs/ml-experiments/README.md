## 分類ガイド

ML改善作業は以下のように分類して記録する。

| 分類 | 内容 | 例 |
|------|------|-----|
| **特徴量エンジニアリング** | モデルに与える入力を改善 | 脚質特徴量の追加 |
| **ハイパーパラメータ調整** | 学習方法の調整 | num_leaves, learning_rate変更 |
| **モデルアーキテクチャ** | アルゴリズムや構造の変更 | デュアルモデル導入 |
| **データ基盤** | データソース・カバレッジの変更 | JRA-VAN移行 |
| **特徴量選択** | 不要な特徴量の削除 | 重要度低の特徴量除去 |
| **検証方法** | 学習/テスト分割の改善 | Cross-validation導入 |

## 重要な学び（横断的）

### 効果が大きかったもの
1. **JRA-VAN IDでの100%マッチ** (v3.0): AUC +0.03は最大の改善
2. **デュアルモデル設計** (v2): Value Bet戦略の基盤
3. **prev_race_popularity** (v3.1): 凡走予測仮説を実証、Model B重要度3位
4. **調教詳細データ統合** (v3.3): Model B AUC +0.0046、VB全ギャップでROI向上
5. **事前オッズ統合** (v3.4): データリーク解消しつつAUC維持、単勝gap>=4で黒字(102.5%)
6. **3-way split + ECE** (v3.5): テスト汚染除去で真の性能を計測、ECE<0.005でキャリブレーション優秀
7. **Win/Placeデュアル** (v5.0): 正確なEV計算を実現、Win AUC 0.8376
8. **33ラップ** (v5.1): ECE全モデル改善、Win予測に特に有効 (prev_race_lap33=22,270)
9. **降格ローテMARKET特徴量** (v5.2b): Model WV AUC +0.0013、市場に見えにくい信号
10. **ベイズ平滑化 = MARKET** (v5.4): Place VB gap>=5 ROI 148.2%（全バージョン最高）。MARKET分類でA-B乖離最大化
11. **IsotonicRegression** (v5.4.1): Win ECE 0.13→0.003（-97%）、全4モデルKelly適格化
12. **marginフィルタ** (v5.5): gap>=5でmargin無し106% → margin<=1.2で119.9%（**+13.9pp、最大の単発改善**）

### 期待ほど効かなかったもの
1. **脚質特徴量**: corners正規化だけでは脚質の本質を捉えきれない
2. **consumption_flag**: 発火条件が厳しく（RPCI<=46 & 21日以内）、該当数が少ない
3. **ペース特徴量（個々）**: 単体では重要度低。ただし集団でAUCを底上げ
4. **個別調教特徴量**: 重要度は低いが、集団効果でModel B AUC +0.0046
5. **CK_DATA調教 in Model B** (v4.1): AUC改善するがROI大幅悪化。MARKET扱いが正解
6. **余力ラップ8特徴量** (v5.2): ROI全gap帯で低下。上位3のみ再実験を検討
7. **降格ローテ個別パターン** (v5.2b): is_koukaku_*の重要度は低い。koukaku_rote_countのみ中位
8. **NLP辞書拡充** (v5.3b): 曖昧語追加でROI低下。少数精鋭が正解
9. **Place VB ROI全般** (v5.5): bet_engine全プリセットでPlace ROI<100%。Win-onlyが最適

### ハイパラの知見
- `feature_fraction`: 特徴量増加時は上げる（0.7->0.8）。各木が新特徴量を拾えるように
- Model B `num_leaves`: 市場系除外で情報量減のため、表現力を上げる（63->127）
- `min_child_samples`/`reg_lambda`: num_leaves増加時は過学習防止で上げる
- `scale_pos_weight=5.0` for Win models: is_win ~6.5% 不均衡対応

### 調教データの知見 (v3.3)
- **追い切りタイムは正規化なしでも効果あり**: コース別・馬場別の補正で更に向上の余地
- **Model B iterが575に延伸**: 調教データが新しい学習シグナルを提供、過学習していない
- **cyokyo_detail構造**: sessions[], oikiri_summary, rest_period をkb_extに埋め込み

### 事前オッズ・データリークの知見 (v3.4)
- **確定->事前オッズ切替のコストは極小**: Model A AUC -0.0001のみ。リーク解消の恩恵が大きい
- **単勝ROIの閾値はgap>=4**: 複勝(128.5%)に加え、単勝(102.5%)も黒字。gap>=5なら単勝125.3%
- **mykeibadb時系列オッズは実用的**: テスト期間100%カバレッジ、フォールバック設計で堅牢

### 特徴量設計の知見 (v4.0)
- **KB印はMARKET扱いが妥当**: mark_point/aggregate_mark_pointは記者印->オッズ高相関
- **末脚ピーク+展開不利の二面検出**: best_l3f_last5（能力上限）とl3_unrewarded_rate_last5（巻き返し候補）
- **着順安定度は馬タイプの数値化**: finish_std_last5で安定型/ムラ型を区別
- **盛り返し強度は戦意の指標**: worst_corner_position - finish_positionで「差す」傾向を定量化

### 4モデル体系の知見 (v5.0)
- **Win AUC > Place AUC**: 1着は明確なシグナルで識別しやすい（0.8376 vs 0.8243）
- **Win ECEが高い（0.13）**: scale_pos_weightがキャリブレーションを崩す。レース内正規化で対応
- **AUC改善 != ROI改善**: Model B精度向上が市場との乖離を縮めてVB効果を殺す

### 33ラップ・降格ローテの知見 (v5.1-v5.2b)
- **33ラップはコース物理特性**: 市場が完全に織り込めていない情報
- **降格ローテはValue系モデルに効く**: Model WV AUC +0.0013だがVB ROI低下あり
- **特徴量一括追加のリスク**: v5.2で8特徴量まとめて追加 -> ノイズ増。小ロットが安全

### NLP・ベイズ平滑化・キャリブレーションの知見 (v5.3-v5.4.1)
- **NLP辞書は少数精鋭**: 拡充版よりシンプル版がROI高い。曖昧語はノイズ
- **Feature importance低くてもVB ROI改善可能**: コメント特徴量はModel B最下位だがPlace VB +4.9pp
- **ベイズ平滑化 = MARKET**: smoothed rateをModel Bに入れるとA-B乖離縮小でVB ROI低下
- **IsotonicRegression**: valセットでfit、testに適用。ランキングにはraw、EV/Kellyにはcalibrated

### 着差回帰・bet_engineの知見 (v5.5)
- **着差回帰はVB ROIで分類を上回る**: Reg B gap>=5 ROI 91.4% vs Cls B 81.6%(+9.8pp)
- **margin filter = 最大の単発改善**: +13.9pp。着差予測は市場乖離検出の精度を大幅に向上
- **Win-only > Win+Place**: Place ROI<100%が全体を引き下げる
- **margin 1.2が最適**: 1.0は厳しすぎ、1.5は緩すぎ、1.2が甘辛バランス最適
- **均一買いROI ≠ Kelly ROI**: Kelly Place betsが大きくなり全体ROI低下するケースあり

### 評価基盤の知見 (v3.5)
- **3-way splitは必須**: v3.4以前のAUCは早期停止による楽観バイアスを含んでいた。v3.5が真の性能
- **ECE < 0.005は優秀**: 確率予測がそのまま実際の確率に近く、EV計算の信頼性が高い
- **NaN処理はLightGBMネイティブが正解**: fillna(-1)より理論的に正確
- **複勝gap>=3が最高ROI(117%)**: 純粋テスト評価で全バージョン最高。実運用の中核戦略として信頼できる
