# マクリ・ペースクラッシャーがレースに与える影響

> 起点: 「マクリ経験やマクリを使う騎手が突如先行してペースを乱す"ペースクラッシャー"になりそうな馬がもたらす影響、ペースクラッシャーが現れたレースの影響について考察したい」

---

## 1. マクリとは何か

```
マクリ（捲り）:
  レース中盤〜後半にかけて、後方から一気にポジションを上げる戦法。
  通常の「差し」が直線で末脚を使うのに対し、
  マクリは3-4コーナーの時点で既に前に取り付く。

  特徴:
  - 3角あたりから大外を回して一気にポジションUP
  - 通過順位が「10-10-4-2」のような急激な変動パターン
  - 後続の馬を「置いていく」動き
  - 逃げ馬・先行馬にとっては突然のペースアップ

通常の差し: 1角10番手 → 2角10番手 → 3角9番手 → 4角7番手 → ゴール3着
マクリ:      1角10番手 → 2角10番手 → 3角5番手 → 4角2番手 → ゴール1着
                                      ↑ ここで急激にポジションUP
```

## 2. ペースクラッシャーとは何か

### 定義

```
ペースクラッシャー:
  レース序盤〜中盤で本来のペース以上にペースを上げる馬。
  意図的な場合と非意図的な場合がある。

■ タイプ1: マクリ型ペースクラッシャー（中盤〜後半）
  通常は中団〜後方にいる馬が、突然前に行く。
  先行集団は追いかけるか、そのまま行くか判断を迫られる。
  → 先行集団のペースが上がる → 全体のペースが崩壊

■ タイプ2: 逃げ型ペースクラッシャー（序盤）
  先頭でオーバーペースで飛ばす馬。
  他の先行馬がつられてペースが上がる。
  → ハイペースで先行勢が総崩れ → 差し・追込有利

■ タイプ3: 突如先行型（最も予測困難）
  通常は差し・追込脚質なのに、今回は騎手の判断で先行する。
  他の先行馬にとっては「想定外のメンバー」が前にいる。
  → ペースの読みが狂う → 展開が荒れる

★ ユーザーの問いの核心はタイプ1とタイプ3の混合:
  「マクリ経験がある馬（= 通常は後方だが前に行く能力がある）が、
   突如して先行してペースを乱す」ケース。
```

### ペースクラッシャーが生まれる条件

```
■ 馬の特性:
  - スピード能力が高いが、気性や折り合いの問題で後方にいることが多い
  - 「行く気になれば行ける」馬（マクリ実績あり）
  - 距離短縮で、前走より前に行ける体力的余裕がある

■ 騎手の意図:
  - 「展開が向かない」と判断し、自ら展開を作りに行く
  - 具体的な騎手: マクリを使うことで知られる騎手がいる
    （ムーア、デットーリ等の外国人騎手、日本人では田辺、松山等）
  - 逃げ馬が手薄なレースで「行った者勝ち」と判断
  - 目標馬を潰すための戦術（チームプレー）

■ レース条件:
  - スローペースが予想される → マクリが決まりやすい
  - 逃げ馬が弱い → 先頭を奪いやすい
  - 外回りコース → 大外マクリが物理的に可能
```

## 3. ペースクラッシャーがレースに与える影響

### 3-1. 先行集団への影響

```
ペースクラッシャーが中盤で前に行くと:

① 逃げ馬のペース判断が狂う
   逃げ馬は「後ろとの差」を感じてペースを調整している。
   突然後ろから馬が来ると:
   - 「抜かれまい」とペースを上げる → オーバーペース → 失速
   - または冷静に行かせる → しかしペースは上がっている

② 先行馬の消耗
   先行馬はマクリに「つられて」動いてしまう。
   - 追いかける → エネルギー消費 → 直線で脚が残らない
   - 動かない → しかし精神的に焦る → リズムが崩れる

③ 「想定ペース」の崩壊
   レース前にスローペース予想 → 実際はハイペース
   → スロー展開での好走を見込んでいた先行馬が壊滅

結果: 先行集団は「共倒れ」のリスクが増大。
     ペースクラッシャー自身も消耗するが、
     「道連れ」で先行馬を潰す効果がある。
```

### 3-2. 後方待機組への影響

```
■ 差し・追込馬にとっての影響:

好影響:
  ① ペースが上がる → 先行勢が止まる → 差しが届く
  ② 先行集団のラインが乱れる → 進路が開きやすい
  ③ 全体の上がりタイムが遅くなる → 末脚の差が活きやすい

悪影響:
  ① ペースクラッシャーが外を回ってマクリ
     → 外の差し馬の進路を塞ぐ場合がある
  ② ペースが上がりすぎて、差し馬もつられて動いてしまう
     → 全体が消耗するレース → スタミナが問われる
  ③ ペースクラッシャー自身が先行集団を潰して
     そのまま残る → 差し馬が想定した「先行崩壊」が
     中途半端に終わる

★ 総合すると:
  ペースクラッシャーの出現は差し・追込馬にとって
  平均的にはプラス。ただし確実にプラスとは限らない。
```

### 3-3. ペースクラッシャー自身の結果

```
マクリで前に行った馬自身はどうなるか:

■ 好走パターン:
  - スローペースのレースで外からマクリ
    → 先行集団が楽をしていたところにプレッシャー
    → 先行馬が対応できず失速
    → マクリ馬が先頭〜好位で直線に入る
    → そのまま押し切り or 余力で粘る

■ 凡走パターン:
  - ハイペースのレースでマクリ
    → 自分自身もハイペースの影響を受ける
    → 3-4コーナーで前に行くエネルギーが加算
    → 直線で止まる（「行って止まる」パターン）

■ 「道連れ」パターン:
  - マクリで先行馬を潰すが自身も止まる
    → 先行集団とマクリ馬の両方が失速
    → 後方で脚を溜めていた馬が漁夫の利
    → 結果的に「穴馬」の好走を演出

★ マクリの成功率:
  一般的にマクリは「ハイリスク・ハイリターン」の戦法。
  成功すれば圧勝もあるが、失敗すると大敗。
  → 着順分布の分散が大きい = [一発型](horse_performance_distribution.md)
```

## 4. 予測困難性: なぜペースクラッシャーはAIにとって難題か

### 4-1. 「今回マクリを使うか」は予測が極めて難しい

```
マクリは「戦法」であり「能力」ではない:
  - 同じ馬・同じ騎手でも、レースによってマクリする/しない
  - 騎手のレース中のリアルタイム判断に依存
  - 「展開を見て判断」されるため、事前予測が困難

予測に使える情報:
  ✅ 過去走でのマクリ実績（頻度）
  ✅ 騎手のマクリ使用頻度
  ✅ 今回の逃げ馬の弱さ（逃げ馬が弱いとマクリしやすい）
  ✅ ペースがスローになりそうか（スローならマクリが決まりやすい）
  ✅ コースの形態（外回りはマクリしやすい）

予測に使えない情報:
  ❌ 騎手の当日の作戦
  ❌ 馬の当日のテンション
  ❌ レース中のリアルタイム判断

→ 「マクリが起こる確率」は推定できるが、
  「起こるか起こらないか」は二値的に予測できない。
  これは展開シミュレーションの [レベル2-3](race_scenario_simulation.md) の問題。
```

### 4-2. ペースクラッシャーの存在はレースの「分散」を増大させる

```
ペースクラッシャーがいる可能性があるレース:
  → 展開のシナリオが増える
  → 結果の不確実性が増大
  → 予測の信頼度が低下

これは前回の [展開リスク指数](smooth_trip_without_simulation.md) と直結する:
  ペースクラッシャー候補がいる → 展開リスクが高い
  → このレースは「見送り」候補 or EV閾値引き上げ

★ 予測が難しいものを無理に予測するより、
  「予測が難しいレースを避ける」方が合理的。
```

## 5. JRA-VANデータから何が検出・推定できるか

### 5-1. マクリ実績の検出

```
マクリの定義（通過順位パターンから検出）:

条件:
  ① 3角通過順位 - 1角通過順位 ≤ -4（4つ以上ポジションUP）
  ② かつ 3角通過順位 ≤ 5（前方に取り付いている）
  ③ かつ 1角通過順位 ≥ 8（スタートは後方だった）

例:
  通過順位 10-10-4-2 → (4-10) = -6 ≤ -4, 3角4 ≤ 5, 1角10 ≥ 8 → マクリ
  通過順位 8-8-6-5 → (6-8) = -2 > -4 → マクリではない（徐々に前進）
  通過順位 12-12-3-1 → マクリ（大捲り）

★ 閾値は仮。過去データで「マクリと判定されたレース」を
  実際のレース映像と照合して閾値を調整すべき。

特徴量化:
  マクリ率 = 過去N走でマクリと判定された回数 / N
  マクリ強度平均 = マクリ時の(1角順位 - 3角順位)の平均
```

### 5-2. 騎手のマクリ傾向

```
騎手別のマクリ使用頻度:
  全騎乗中のマクリ率を計算。

  仮に:
  騎手A: 騎乗300回中マクリ15回 = 5% → マクリ騎手
  騎手B: 騎乗300回中マクリ2回 = 0.7% → ほぼ使わない

  → 「マクリ率が高い騎手 × マクリ実績がある馬」の組合せは
    ペースクラッシャーになる確率が高い。

騎手乗り替わりの影響:
  前走は「控える騎手」→ 今回は「マクリ騎手」に乗り替わり
  → 戦法が変わる可能性 → 展開予測の不確実性UP
  ★ これは非常に重要な観点。騎手の乗り替わりは
    単なる「騎手の腕の差」だけでなく、
    「戦法の変化」も意味する。
```

### 5-3. ペースクラッシャー候補の事前特定

```
以下の条件が重なる馬はペースクラッシャー候補:

① マクリ率が高い（過去走で10%以上）
② 今回の騎手がマクリを使う騎手
③ レースの逃げ馬が弱い or いない
④ スローペースが予想される
⑤ 外回りコース（マクリが物理的に可能）
⑥ 前走からの距離短縮（スタミナに余裕がある）

全てが揃う必要はないが、複数該当すれば確率は上がる。

特徴量:
  pace_crasher_prob = f(マクリ率, 騎手マクリ率, 逃げ馬弱さ,
                         予想ペース, コース形態, 距離変化)
```

## 6. ペースクラッシャーが現れたレースの事後分析

### 「ペースクラッシャーレース」の特徴

```
ペースクラッシャーが出現したレースを過去データから特定し、
その影響を統計的に分析する。

特定方法:
  レース中にマクリ判定（セクション5-1の条件）に該当する馬がいたレース

分析項目:
  ① ペースクラッシャーレースの先行馬の成績
     → 先行馬の着順は平均より悪いか？
     → 先行馬の上がり3Fは平均より遅いか？

  ② 差し・追込馬の成績
     → 差し・追込馬の着順は平均より良いか？
     → 差し・追込馬の回収率はどうか？

  ③ 結果の荒れ度
     → 1番人気の着順は平均より悪いか？
     → 配当は平均より高いか？
     → 三連単の万馬券率はどうか？

  ④ ペースクラッシャー自身の成績
     → マクリ成功率はどの程度か？
     → 成功時の着順は？失敗時の着順は？

仮説: ペースクラッシャーが出現したレースは、
     先行馬が平均的に凡走し、差し馬が好走する。
     結果が荒れやすく、高配当になりやすい。
```

### 「道連れ効果」の定量化

```
ペースクラッシャー自身の着順と、先行馬の着順の関係:

パターン1: クラッシャー好走 + 先行馬壊滅
  → クラッシャーのマクリが成功。先行馬がペースについていけず失速。
  → 差し馬も好走（ハイペースの恩恵）

パターン2: クラッシャー凡走 + 先行馬壊滅
  → 「道連れ」パターン。クラッシャーも先行馬も共倒れ。
  → 後方で脚を溜めていた馬が上位独占 → 大荒れ
  ★ このパターンが最も配当が高くなる

パターン3: クラッシャー凡走 + 先行馬好走
  → マクリが不発。先行馬はペースを崩されず。
  → 展開変わらず → 普通のレース

→ パターン2の頻度と配当を分析すれば、
  「ペースクラッシャーがいるレースで穴馬を狙う」戦略の
  期待値が計算できる。
```

## 7. 特徴量設計

### マクリ・ペースクラッシャー系の新特徴量

```
■ F12: マクリ率（過去N走）
  定義: 過去N走でマクリ判定に該当した割合
  計算: count(マクリ判定) / N
  用途: この馬がマクリを使うタイプかの指標

■ F13: 騎手マクリ率
  定義: 当該騎手の過去1年の全騎乗でのマクリ使用率
  計算: count(騎手の騎乗でマクリ判定) / total_rides
  用途: この騎手がマクリを選択する確率

■ F14: ペースクラッシャー候補度
  定義: マクリ率 × 騎手マクリ率 × (条件補正)
  計算:
    base = horse_makuri_rate × jockey_makuri_rate
    条件補正:
      × 逃げ馬弱さ指数（逃げ馬のクラス補正能力値の逆数）
      × コース外回りフラグ（外回り=1.5, 内回り=1.0）
      × 距離短縮フラグ（短縮=1.3, 延長=0.8, 同距離=1.0）
  用途: 今回のレースでこの馬がペースクラッシャーになる確率

■ F15: 同レースのペースクラッシャー候補数
  定義: 同レースの出走馬中、F14が一定閾値以上の馬の数
  用途: このレースの展開リスク（ペースが乱される確率）

■ F16: ペースクラッシャー脅威度（先行馬向け）
  定義: 先行脚質の馬にとって、同レースにペースクラッシャー候補が
        いることによる脅威の大きさ
  計算: max(同レースの各馬のF14) × (自分が先行脚質なら1.5, そうでなければ0.5)
  用途: 先行馬の「ペースを乱されるリスク」

■ F17: 騎手乗り替わり×マクリ傾向変化
  定義: 前走騎手のマクリ率と今回騎手のマクリ率の差
  計算: jockey_makuri_rate_current - jockey_makuri_rate_previous
  用途: 戦法変化の可能性。正=マクリ傾向が増す乗り替わり
```

### 事後分析用の特徴量（過去走のレース質を評価）

```
■ F18: 前走レースのペースクラッシャー影響度
  定義: 前走のレースでマクリが発生したか、それによる影響
  計算:
    前走にマクリ馬がいた場合:
      影響度 = マクリ馬の(1角順位 - 3角順位)の絶対値 / 頭数
    マクリ馬がいなかった場合:
      影響度 = 0
  用途: 前走が「荒れた展開」だったかの指標。
        前走の結果を評価する際の補正に使える。
        （荒れた展開で凡走 → 度外視すべき凡走の可能性）

■ F19: 前走レースでの自分の被害度
  定義: マクリが発生したレースで、自分がどれだけ影響を受けたか
  計算:
    自分が先行脚質 && マクリ馬がいた → 被害度 = 高
    自分が差し脚質 && マクリ馬がいた → 被害度 = 低〜なし
  用途: 前走凡走の原因がペースクラッシャーにあるなら、
        次走で巻き返し候補として評価できる。
```

## 8. 馬券戦略への応用

### 戦略1: ペースクラッシャー候補がいるレースで差し馬を狙う

```
条件:
  - F15（ペースクラッシャー候補数）≥ 1
  - 先行馬が人気上位を占めている
  - 差し・追込馬が軽視されている（人気薄）

期待:
  ペースクラッシャーが発動 → 先行馬壊滅 → 差し馬が浮上
  → 差し馬のオッズが実力以上に高い → 期待値あり

リスク:
  ペースクラッシャーが発動しない確率もある
  → 発動しなければ先行有利のレース → 差し馬は凡走
  → 期待値計算にはペースクラッシャー発動確率を織り込む必要

→ これはまさに [展開シミュレーション](race_scenario_simulation.md) の
  レベル2（シナリオベース）の応用。
  シナリオ1: マクリ発動（確率P1）→ 差し馬有利
  シナリオ2: マクリ不発（確率1-P1）→ 先行馬有利
  → 加重平均で期待値を計算
```

### 戦略2: ペースクラッシャーレースを「荒れレース」として高配当を狙う

```
ペースクラッシャー候補がいるレースは:
  → 結果の分散が大きい
  → 人気薄の好走確率が上がる
  → 三連単の配当が高くなりやすい

戦略:
  F15 ≥ 1 のレースでは:
  - 三連単の買い目に「差し・追込の穴馬」を多めに組み込む
  - ワイド・馬連では「先行馬×差し馬」のbox買い
  - 単勝では差し馬の単勝を追加

★ ただし「荒れレース」は予測精度自体が低いため、
  [展開リスク指数](smooth_trip_without_simulation.md) が高い
  → EV閾値を引き上げる、の判断も並行して行う。
```

### 戦略3: ペースクラッシャーの被害を受けた馬の次走を狙う

```
これが最も実用的かつ高い期待値が見込める。

条件:
  - 前走でマクリが発生（F18 > 0）
  - 自分は先行脚質で前走凡走（着順8着以下）
  - しかし前走の上がり3F自体は水準レベル
    （完全に止まったわけではない → 能力はある）
  - 今回のレースにペースクラッシャー候補がいない（F15 = 0）
  - オッズが前走凡走を反映して高い（過小評価）

期待:
  「前走は展開の犠牲になっただけ。今回は平穏な展開→好走」
  → [前走不利検出](smooth_trip_without_simulation.md) の一種。

★ これは展開を「予測する」のではなく、
  過去の展開の影響を「事後評価する」アプローチ。
  データで完結し、精度も高い。
```

## 9. 将来的な発展: マクリの意思決定モデル

```
長期的な研究テーマとして:

「この騎手×この馬が、このメンバー構成で、マクリを使う確率」
をモデル化できれば、展開予測の精度が大きく向上する。

入力:
  - 馬のスピード能力（マクリに必要な加速力）
  - 騎手のマクリ傾向
  - レースのペース予想（スローなほどマクリしやすい）
  - 逃げ馬の強さ（弱いほどマクリが決まりやすい）
  - コース形態（外回りはマクリしやすい）
  - 馬の疲労度（元気なほどマクリできる）

出力:
  P(マクリ発動) = 0.15（15%の確率でマクリする）

→ この確率を展開シミュレーション（レベル2-3）の
  シナリオ定義に使う。

ただしこれは高度な実装であり、優先度は低い。
まずは:
  1. マクリの事後検出（F12-F13）
  2. ペースクラッシャー候補の特定（F14-F15）
  3. 被害を受けた馬の次走分析（F18-F19）
の順に実装するのが現実的。
```
