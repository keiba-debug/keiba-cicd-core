## 使用する6+1テーブル

| # | テーブル | 用途 |
|---|---------|------|
| 1 | レース結果テーブル | メインの訓練データ。着順→「3着以内」二値分類 |
| 2 | レース情報テーブル | 距離・天気等のマスタ情報 |
| 3 | 馬の過去成績テーブル | 馬ごとの全出走履歴 |
| 4 | 馬の基本情報テーブル | 馬の属性情報 |
| 5 | 血統テーブル | 5世代分の血統 |
| 6 | 払い戻しテーブル | 回収率シミュレーション用 |
| (+1) | 出馬表 | 当日予測時にスクレイピング |

### v1との差分
- **馬の基本情報テーブル** が新規追加（v1にはなかった）
- **レース情報テーブル** がレース結果テーブルから分離

## 処理フロー

### 学習時
```
スクレイピング（6テーブル）
  ↓ 各テーブルの前処理
  ↓ テーブル2,3,4をテーブル1に結合
    ※ 馬の過去成績は「直近5走の集計」等を計算してから結合
  ↓ 血統テーブルを結合
  ↓ カテゴリ変数処理
  ↓ 訓練/テスト分割
  ↓ モデル学習
  ↓ テストデータ + 払い戻しテーブルで回収率シミュレーション
```

### 予測時（レース当日）
```
前日:
  - 出走馬確定
  - 馬の過去成績・血統を最新に更新

当日（馬体重発表後）:
  - 出馬表スクレイピング
  - 訓練データと同じ加工処理
  - 学習済みモデルに投入
  → 予測スコア（3着以内確率）出力
```
