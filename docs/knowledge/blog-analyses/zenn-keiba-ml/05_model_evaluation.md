## 評価の全体像

```
LGBMClassifier
  ↓ predict_proba() → レース内標準化スコア
  ↓ threshold で賭ける/賭けないを判定
  ↓ gain() で threshold を変化させて回収率をプロット
  ↓ 標準偏差 + シャープレシオで「安定した利益」を評価
```

---

## predict_proba の特徴: レース内標準化

```python
standard_scaler = lambda x: (x - x.mean()) / x.std(ddof=0)
proba = proba.groupby(level=0).transform(standard_scaler)
```

- **生の確率ではなく、レース内偏差値に変換**
- 「レース内偏差値」= レースごとの相対評価
- threshold=2.0 は「そのレース内でスコアが平均+2σ以上」を意味
- 狙い目のthreshold: 単勝で2.5〜3.0あたり

## 馬券種別シミュレーション

### 対応馬券
| 馬券 | BOX | 流し |
|------|-----|------|
| 単勝 | ○ | - |
| 複勝 | ○ | - |
| 馬連 | ○ | ○ |
| 馬単 | ○ | ○ |
| ワイド | ○ | ○ |
| 三連複 | ○ | - |
| 三連単 | ○ | ○ |

### BOX方式
- threshold超えの馬全員で全組み合わせ
- 問題: threshold超え馬が少ないとbet数ゼロ

### 流し方式
- threshold超え馬が1頭 → 軸にしてスコア上位N頭を相手
- threshold超え馬が2頭以上 → BOXと同じ
- 三連単流し: 軸2頭 + 相手上位7頭（城崎哲『AI競馬』参考）

### シミュレーション結果の傾向
- **単勝が最もシンプルで回収率が出る**
- 馬連・馬単BOXは単勝に劣る
- **三連単流し（軸2頭）でthreshold 1.5付近で100%超え**
- 推奨戦略: 「スコア2.5以上→単勝、それ以外→threshold 1.5で三連単流し」

## 標準偏差とシャープレシオ

### 回収率の標準偏差

```
σ(r) = √n × σ(X) / b
```
- X: 1レースの払い戻し金額
- n: レース数
- b: 購入金額

```python
std = np.std(return_list) * np.sqrt(len(return_list)) / n_bets
```

### シャープレシオ

```
シャープレシオ = (回収率 - 1.0) / σ(回収率)
```
- 無リスク資産 = 競馬をやらない = 回収率100%
- **高いほど「安定して稼げる」**
- 単勝 > 単勝適正回収値（σが小さくても回収率の差で単勝が勝つ）

### 単勝適正回収値
- `tansho_return_proper`: 払い戻しが一定になるよう `金額 = 1/オッズ` で賭ける
- 高オッズ馬に引っ張られない安定した指標
- ただしシャープレシオでは普通の単勝に劣る

## 設計上の注意点（著者の教訓）

### bet関数の分離原則
- 的中判定は**bet関数の中でのみ**return_tableを参照
- 理由1: return_tableの情報リークを防ぐ（著者が実際にやらかした）
  - 「レースが荒れるかどうか事前にわかった状態で掛け金を決めてしまう」ミス
- 理由2: 「一つの関数には一つの役割」の原則

## 参考になるポイント

1. **★ シャープレシオ** — 回収率だけでなく「安定性」を評価する指標。うちにも導入価値あり。ROIだけ見てると分散の大きい戦略を選びがち
2. **★ レース内標準化** — うちのAR偏差値（`50 + 10*(AR-mean)/std`）と本質的に同じ発想。確率の相対評価
3. **threshold sweepのgain関数** — thresholdを連続的に変化させて回収率カーブを描く。うちのgap閾値sweep分析と同じ
4. **bet関数の分離設計** — return_tableの情報リーク防止。うちのbacktest_bet_engineでも同じ設計思想
5. **三連単流し（軸2頭+相手7頭）** — 現時点のうちは単複のみだが、将来の馬券種拡大時に参考
6. **n_horsesの重要度1位** — うちにもn_horsesあり。base_features.pyで実装済み

## シャープレシオ導入メモ

うちで導入する場合:
- `backtest_bet_engine.py` の結果からレースごとのP&Lを算出
- `σ(r) = √n × σ(X) / b` で回収率の標準偏差を計算
- `SR = (ROI - 1.0) / σ(r)` でシャープレシオ
- プリセット比較時の判断基準に追加できる
