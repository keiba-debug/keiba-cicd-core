# 競馬AIでポルシェを買う話（2019年の馬券の税金 -600万円）

- **出典**: Matsukaze.AI「競馬AIでポルシェを買う話（2019年の馬券の税金 -600万）」 http://matsukaze.ai/?p=890
- **日付**: 2025-02-11（元記事: 2020-02-26 / 更新: 2020-10-09）

## 要点

### 2019年の納税額

- **所得税**: 約450万円
- **住民税**: 約150万円
- **合計**: 約600万円
- ふるさと納税で70〜80万円を先払い済み
- 馬券の利益1,700万円に対して約35%の税負担

### 一時所得 vs 雑所得（最大の論点）

#### 一時所得の場合（致命的シナリオ）

- 払戻金1億4,000万円が一時所得
- 当たり馬券の購入金額（推定約1,000万円）と特別控除50万円を控除
- 課税所得 ≈ 6,500万円 → **税金約3,000万円**
- 利益1,700万円に対して3,000万円の税金 → **1,300万円の負債**
- **自己破産も原則不可能（税金は非免責債権）**

#### 雑所得の場合（Matsukaze氏の主張）

- 外れ馬券を必要経費として控除可能
- 利益1,700万円に対して課税 → 税金約600万円
- 実質利益 ≈ 1,100万円

### 雑所得の要件（国税庁通達の分解）

Matsukaze氏が通達の各要件を自分のケースに照合:

| 通達の要件 | Matsukaze氏の状況 | 争点リスク |
|---|---|---|
| 馬券を自動購入するソフトウエア | 松風は自動購入ソフト | **なし** |
| 独自の条件設定と計算式 | 統計学に基づく計算式 | 低 |
| 予想確度×配当率の購入パターン | EVベースの購入パターン | **なし** |
| ほぼ全てのレースで購入 | 3,452レース中2,437レース（**約70%**） | **あり** |
| 年間を通じて多数の馬券を購入 | 年間**24,459点**を購入 | やや**あり** |
| 年間を通じて多額の利益 | **+1,700万円** | やや**あり** |
| 回収率が期間総体で100%超 | **114%**（複利ベースで5,666%） | **なし** |
| 客観的に明らか | Twitter記録+通帳 | **なし** |

### 3つの争点

1. **「ほぼ全て」の定義**: 70%が「ほぼ全て」か？
2. **「多数」の定義**: 24,459点が「多数」か？
3. **「多額」の定義**: 1,700万円が「多額」か？

### 判例の援用

- 2019年10月の東京地裁判決: 年間数千万円の購入でも「営利目的で継続的」と認定
- 税務署は「最高裁判決（30億円）より小規模」と主張したが退けられた
- Matsukaze氏（年間1.24億円購入）はこの判例を下回る規模ではない

## AI開発への示唆

### 1. 税務設計はAI設計の一部 ★★★

> この通達ありきで松風を設計し、この要件を満たせるから、運用を開始しています

- **AIの設計段階から税務要件を織り込む必要がある**
- 雑所得の要件を満たすための設計条件:
  - 自動購入ソフトウエアであること
  - 計算式に基づく購入パターン
  - 年間を通じてほぼ全レースで購入
  - 年間通算で回収率100%超
  - 客観的な記録（ログ）

### 2. 「ほぼ全てのレース」への設計対応

- 2019年JRA全3,452レース中2,437レース（70%）で購入
- **より安全にするなら80%以上を目指すべき**
- 設計上の含意:
  - EV閾値を下げて購入レース数を増やす
  - 「見送り」を減らす方向のチューニング
  - これは006(5週目)の「EV>1.0を全部買え」と整合

### 3. 記録・証跡の自動保存

- Twitter投稿（締切前タイムスタンプ）= 客観性の証拠
- 通帳記帳との整合
- **我々のシステムでも必要な記録**:
  - 購入前の買い目ログ（タイムスタンプ付き）
  - AIの予測値・EV計算の根拠ログ
  - 購入結果・払戻結果のログ
  - 全てを改ざん不可能な形で保存

### 4. 税引後リターンの最適化

- 雑所得の場合: 利益の約35%が税金
- **税引後の回収率** = 1 + (回収率 - 1) × 0.65
  - 回収率114% → 税引後回収率: 約109%
- 経費計上可能な項目:
  - サーバー費用、通信費
  - PC・モニター（減価償却）
  - データ購入費（JRA-VANなど）
  - 税理士費用
- **ふるさと納税の活用**: 70〜80万円分を先払い

## 独自考察

### 税務リスクをAI設計に織り込むフレームワーク

```
設計チェックリスト（税務要件）:
  [✓] 自動購入ソフトウエアである
  [✓] 独自の条件設定と計算式がある
  [✓] 確度×配当率に基づく購入パターン
  [ ] 年間80%以上のレースで購入
  [✓] 年間通算で回収率100%超
  [ ] 全購入記録を客観的に保存
  [ ] 税理士との事前相談
```

### 購入レース比率の最適化

- 70%（Matsukaze氏2019年）は争点リスクあり
- 100%に近づけるほど税務的には安全だが、EV < 1.0のレースも買うことになる
- **最適点**: EV > 0.95程度まで購入範囲を広げれば、購入レース比率80%+を達成できる可能性
- 小額のEV < 1.0購入は「税務コスト」として許容する判断もあり

### 税制が馬券戦略に与える影響

| 税区分 | 外れ馬券 | 実質税率 | 利益1,700万の場合 |
|---|---|---|---|
| 一時所得 | 経費不可 | 約176%（破産） | -1,300万円 |
| 雑所得 | 経費可 | 約35% | +1,100万円 |
| 差額 | - | - | **2,400万円** |

→ **税区分の違いだけで2,400万円の差**。税務設計はモデル精度と同等以上に重要。

### Matsukaze氏の証拠設計の周到さ

- 運用開始前から税務通達を認識し、要件を満たす設計にした
- Twitterで締切前に買い目を公開 = 客観性の担保
- 「買い目なし」の時もツイート = 後から都合良くデータを消せない証明
- **これは「税務設計」というよりも「証拠設計」**
