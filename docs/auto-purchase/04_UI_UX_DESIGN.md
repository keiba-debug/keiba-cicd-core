# 04. UI/UX設計 — 自動購入状況の見える化

## 1. ユーザとジョブ

| ペルソナ | ジョブ | 成功状態 |
|----------|--------|----------|
| 当日運用者 | 午前に今日の購入方針を決め、午後はレースごとに判断 | 1画面で「次に何をすべきか」が分かる |
| 事後分析者 | なぜ買った/買わなかったかを追跡 | ledger + confirmed_bets + memo が繋がる |
| 緊急時 | すぐ全停止 | 1クリックで `stopped`、理由を残す |

## 2. 情報設計（IA）

```
/bankroll（既存）
  ├─ Fund / Performance（既存）
  └─ Execute（既存）… 買い目選択・FF・買い確定

/auto-purchase（新規）★本設計の中心
  ├─ 今日のパイプライン（デフォルト）
  ├─ イベントログ
  └─ 設定（モード・締切オフセット・通知）

/predictions（既存・拡張）
  └─ JIT Phase1: オッズ乖離・EV転落バッジ

/races-v2/[date]/[track]/[id]（既存・拡張）
  └─ 購入情報モーダル（web-roadmap §1）
```

**推奨**: 新規 `/auto-purchase` を設け、ExecuteTab から「実行状況を見る」リンクで遷移（ExecuteTab は肥大化させない）。

### 3.2 パイプライン・ドット（6段）

| # | 段 | 意味 | 色 |

| 2 | 承 | 人間承認（半自動） | |

| 5 | IP | IPAT最終投票（人手） | |
| 6 | 決 | 精算済み | |

### 3.3 状態バッジ（一覧用）

| 状態 | 表示 | 色 | ユーザーアクション |
|------|------|-----|-------------------|
| SCHEDULED | 待機 | 灰 | — |
| ELIGIBLE | 対象 | 青 | — |
| PENDING_APPROVAL | 承認待ち | 橙 | 承認 / 見送り |
| FF_WRITTEN | FF出力済 | 青 | TARGET取込後チェック |

## 5. ヘッダ・コントロール

### 5.1 モード切替

| モード | UI | 確認ダイアログ |
|--------|-----|----------------|
| manual | グレーアウト（パイプライン停止） | — |
| semi | デフォルト | — |
| auto_ff | 上級者向けバッジ | 「承認スキップします」 |
| stopped | 赤背景 | 解除時に理由入力 |

### 5.2 緊急停止

- ボタン: 常時表示、キーボード `Ctrl+Shift+S`（任意）
- 効果: Orchestrator 停止 + モード `stopped` + イベント記録
- 解除: 管理者のみ、理由必須

### 5.3 「次のアクション」バー

優先度:

1. `FAILED` / `STALE` の修復
2. `PENDING_APPROVAL` で締切 < 10分
3. `AWAITING_IPAT` で締切 < 5分
4. 次の `ELIGIBLE` レース

## 6. 既存画面への拡張

### 6.1 `/predictions`（JIT Phase 1）

| 要素 | 仕様 |
|------|------|
| カラム「オッズ変動」 | 予測時比 %、↑↓ |
| バッジ「見送り推奨」 | ライブ EV < 1.0 |
| 閾値表示 | 「あとX%下落でEV<1」 |
| リンク | 該当レース → `/auto-purchase` ドロワー |

### 6.2 `/bankroll` ExecuteTab

| 追加 | 仕様 |
|------|------|
| リンク | 「コックピットで見る」→ `/auto-purchase?date=` |
| バッジ | 本日 `要対応 N件` |
| FF出力後 | トースト「TARGET取込→IPAT投票→コックピットで完了報告」 |

### 6.3 レース詳細

- web-roadmap §1 購入情報モーダルと連携
- ledger の `SUBMITTED` / `SETTLED` と purchases を突合表示

## 7. 通知（Phase 1+）

| チャネル | トリガ | Phase |
|----------|--------|-------|
| ブラウザ通知 | 承認待ち・締切5分前 | 1 |
| 音 | 緊急停止・FAILED | 1 |
| Windowsトースト | 同上（Tauri/Electron不要、PWA可） | 2 |
| LINE/Slack Webhook | 任意 | 3 |

## 8. モバイル・第二画面

- 最低: 一覧の「要対応」フィルタと承認ボタンがタップ可能
- 横スクロール: パイプラインドットは省略し状態バッジのみ

## 9. アクセシビリティ・誤操作防止

| 対策 | 内容 |
|------|------|
| 二段階承認 | 金額 > レース上限50% で確認モーダル |
| 見送り理由 | プリセット: EV転落 / 上限 / 直感 / その他 |
| 操作ログ | 全ボタン操作を ledger events に記録 |
| 色覚 | 状態は色+アイコン+テキストの三重重複 |

## 12. UX メトリクス（改善のため）

| KPI | 定義 |
|-----|------|
| 承認リードタイム | PENDING → FF_WRITTEN の秒数 |
| IPAT確認漏れ率 | STALE / 全 FF_WRITTEN |
| 見送り率 | SKIPPED / ELIGIBLE |
| 誤購入エスカレーション | 人間が purchases を手修正した件数 |
