## プロンプト A：ビジュアルレビュー（6人共通）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 前回のおさらい
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

宿題③aで全員がキャラクター素材を出した。合計68人。
そのキャラたちのビジュアルを初期生成した。
今回はそのビジュアルを見て評価・調整する。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 今回の宿題：ビジュアルを評価しろ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

添付画像を全て見て、以下を回答してほしい。

### 1. 第一印象
各画像について一言ずつ（「この人は○○に見える」レベルでいい）。

### 2. 好きな画像 TOP5
理由付きで。「このキャラを作品に出したい」と思えるもの。

### 3. 嫌いな画像 / 調整が必要な画像
何がダメか具体的に。「目が同じ」「服が違う」「空気感が合わない」等。

### 4. 目の差別化
現状、女性キャラの目が似すぎている可能性が高い。
各キャラの目はどう違うべきか？以下の項目で指定してほしい：
- 目の形（丸い / 切れ長 / 垂れ目 / つり目）
- 視線の癖（伏し目 / 正面 / 横目 / 遠く）
- 瞳の大きさ（大 / 中 / 小）
- 眉の形（太い / 細い / 困り眉 / まっすぐ）

### 5. 記号アイテムの確認
各キャラの「記号」（象徴するモノ）が画像に正しく反映されているか確認。
足りないもの、変えたほうがいいものを指摘。

### 6. 「別人に見えるか」テスト
自分が出したキャラ同士を並べて、パッと見で区別がつくか確認。
区別がつかないペアがあれば報告。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 出力ルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- アウトプットは必ずMarkdown形式
- 冒頭に名前タグ（【カイ】【ユキ】【レイ】【シュウ】【シノ】【キバ】）
- 全回答をテキストとして出力（「後で出します」禁止）
- 推測箇所は「※推測」と注記
```

## プロンプト B：逆設定（シュウ・キバ専用）

> シュウとキバはリサーチャー。画像から「人間を読む」のが得意分野。
> 元の設定書を渡さず、顔だけから情報を引き出す。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 今回の宿題：絵から人間を読め
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

添付された画像を見ろ。名前も設定も忘れていい。

この顔の人間について、以下を「取材ノート」形式で書け：

### 各画像につき：
- この人間は何歳か
- 何の仕事をしているか（していたか）
- 財布の中身は何か
- 今日何を食べたか
- 競馬場に来た理由は何か
- 誰と一番仲良くなりそうか（他の画像から選べ）
- この人間の一番おいしいエピソード候補

ルール：
- 元の設定と一致している必要はない
- 「この顔からしか出てこない情報」を書け
- 設定書を参照するな。目の前の顔だけを見ろ

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 出力ルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 冒頭に【シュウ】or【キバ】
- 取材ノート形式
- 全画像分を出力すること
```

## プロンプト C：プロンプト調整指示（ユキ専用）

> ユキは画像生成プロンプトの原案者。
> 実際の画像とのズレを見て、修正版プロンプトを出す。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 今回の宿題：画像生成プロンプトを直せ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

添付画像と、前回のプロンプト集（homework_03a_image_prompts.md）を見比べろ。

以下をやってほしい：

### 1. プロンプトと実際の画像のズレ
各キャラについて「プロンプトにはこう書いたが、実際の画像はこうなった」を列挙。

### 2. 修正版プロンプト
以下の問題を解決した修正版を出してほしい：

**目の差別化**（最重要）：
- ソラ：伏し目、少し困った眉、瞳やや大きめ → これが基準
- 他キャラは全員ソラと「違う目」にすること
- 各キャラの目の特徴をプロンプトに明示的に書くこと

**統一感の維持**：
- 線の太さ、色温度、デフォルメ度合いが全キャラで揃うこと
- ソラの既存画像をリファレンスとして指定すること

**記号アイテムの強調**：
- 記号が画面の中で目立つように配置指示を入れること

### 3. 優先生成リスト
「まずこの10人を再生成すべき」という優先順位と理由。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 出力ルール
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 冒頭に【ユキ】
- 修正版プロンプトはそのまま画像生成ツールに貼れる形式で
- 全回答をテキストとして出力
```

## 使い方

### ステップ1：初期ビジュアル生成
- `homework_03a_image_prompts.md` のプロンプトを使って画像を生成
- 既存キャラの画像をスタイル参照として添付
- 生成先：`C:\KEIBA-CICD\キャラクター\オーディション\`
- 命名規則：`kazemachi_[作家ID]_[番号]_[キャラ名].png`

### ステップ2：プロンプト A（ビジュアルレビュー）
- **6人全員** に投げる
- 画像フォルダの中身を添付して渡す
- 各自の視点でレビューさせる

### ステップ3：プロンプト B（逆設定）
- **シュウ・キバ** に投げる（リサーチャー2人）
- 画像だけ渡して、設定書は渡さない
- 「顔から読む」ことでテキスト設定にない情報が出てくる

### ステップ4：プロンプト C（プロンプト調整）
- **ユキ** だけに投げる（元のプロンプトを書いたのがユキなので）
- 画像 + 元プロンプト集を両方渡す
- 目の差別化が最優先課題

### ステップ5：プロンプト D（伏線ビジュアル監査）
- **レイ** だけに投げる（伏線設計者なので）
- 画像 + 自分の03a回答を渡す
- 二層構造の「裏5%」をビジュアルにどう埋め込むか

### ステップ6：プロンプト E（関係性マップ）
- **シノ** だけに投げる（人間観察の専門家なので）
- 画像だけ渡す
- 「全員が同じ空間にいる」ときの自然さを検証
