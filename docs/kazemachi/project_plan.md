# kazemachi — 技術パイプライン・展開計画

> 設計書（聖典）は `PROJECT_DESIGN.md`。ここは技術面とロードマップ。

---

## 1. コンセプト

### 何を作るのか

「競馬×キャラクター×物語」の群像劇コンテンツ。
個性の異なるキャラクターたちが、KeibaCICD の実データをもとに
毎週の競馬を語り、その中で関係性と秘密が少しずつ見えてくる。

### なぜ他にないのか

> フィクションの連載は「確定した物語」を追うだけ。
> この企画は「毎週現実が起きる」。

- 実際のレース結果が物語を動かす
- 誰も結末を知らない
- キャラが「このコースは昔…」と言った翌週に、本当にそのコースで何かが起きる
- それが偶然か計算か、読者にはわからない

## 2. 技術パイプライン

### データフロー

```
KeibaCICD（Python / ML）
    │
    ├─ レースデータ分析
    │   ├── P%/W% 確率推定
    │   ├── Value Bet判定（市場シグナル）
    │   ├── コース特性・展開予測
    │   └── 過去走・騎手・血統データ
    │
    ├─ JSON出力
    │   └── predictions.json / bets.json
    │
    ├─ キャラクター別プロンプト生成
    │   ├── 各キャラが見るデータを制限・変換
    │   └── 生データ → キャラの言葉に翻訳
    │
    └─ 出力
        ├── X投稿（テキスト+1枚絵）
        ├── Note長文
        ├── ショート動画（静止画+字幕）
        └── [将来] フル動画（音声付き）
```

### データ権利対応

IDM・調教コメント等の生データは直接出力NG。
キャラクターが「自分の目で見た感想」として語る形に変換。

| NG | OK |
|----|-----|
| 「IDM 72.5」 | 「このメンバーの中では能力トップクラス」 |
| 調教コメント原文 | 「調教見たけど、あの動きは本気だった」 |
| JRDBの生データ表示 | 「データ的にはかなり過小評価されてる」 |

自前モデル出力（P%/W%/AR/VB判定）は公開OK。

## 5. 画像テイスト

- **画風**: 日常アニメ系。スキップとローファー〜ふつうの軽音部のライン
- **線**: スケッチ、手描きペン、はみ出し、不完全
- **色**: 低彩度、フラット、ウォーターカラーウォッシュ
- **彩度ルール**: 各キャラ1色だけ鮮やかな色を持つ
- **統一感**: 全キャラが同じアーティストに描かれたように見えること
- **生成ツール**: SeeArt (SeedDream 4.0)
- **注意**: 長いプロンプトは入らない。分割生成で対応
