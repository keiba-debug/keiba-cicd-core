# 公開用データページ構想

> 予想TVサイトの一部として、その日のValueBet一覧等を公開する

---

## コンセプト

予想TVのコンテンツ（キャラの予想バトル）と連動して、
実データに基づく推奨馬一覧を公開ページとして提供する。

「キャラが熱く議論してた馬、実際のデータはこれ」→ 信頼性とエンタメの両立

---

## 公開できるデータ / できないデータ

### ✅ 公開OK（自前の分析結果・公開情報）

| データ | 理由 |
|--------|------|
| P%（複勝確率） | 自前MLモデルの出力 |
| W%（勝率） | 自前MLモデルの出力 |
| AR（期待値レシオ） | 自前計算 |
| VB判定（妙味/鉄板等） | 自前分析 |
| market_signal | 自前のオッズ分析結果 |
| オッズ | JRA公開情報 |
| 馬名・騎手名・枠番 | 公開情報 |
| レース名・コース・条件 | 公開情報 |
| キャラ別予想・買い目 | 自前コンテンツ |
| キャラ別成績 | 自前コンテンツ |

### ❌ 公開NG（外部データソースの著作物）

| データ | 理由 |
|--------|------|
| IDM（具体的数値） | JRA-VANデータ |
| 調教コメント原文 | 競馬ブック著作物 |
| JRDB指数・生データ | JRDB著作物 |
| keibabook拡張データ原文 | 競馬ブック著作物 |

### ⚠️ グレーゾーン（表現を工夫すれば可能）

| データ | 変換方法 |
|--------|---------|
| 能力評価 | IDM値ではなく★5段階やランク（S/A/B/C）で表現 |
| 調教評価 | コメント原文ではなく「好調/普通/不安」等の独自評価 |
| 近走成績 | 着順は公開情報なので表示OK |
| 特徴量重要度 | モデルが何を重視してるかは自前情報 |

## 技術的な実装イメージ

### データフロー
```
KeibaCICD (predictions.json / bets.json)
    │
    ├─ public_export.py（公開用フィルター）
    │   ├── NG項目を除外
    │   ├── グレー項目を変換（IDM→ランク等）
    │   └── キャラ別コメントを付与（Claude API）
    │
    ├─ public_data/{date}.json
    │
    └─ Azure Static Web Apps（or Vercel）
        └── 公開ページとしてレンダリング
```

### public_export.py の役割
- predictions.jsonから公開可能なフィールドだけ抽出
- IDM等のNG項目を自動除外
- 能力値を★ランクに変換
- 出力: public_data/{date}.json

### ポイント
- **内部用と公開用は明確に分離する**
- 公開用JSONを生成するスクリプトを1本作れば、あとは自動化できる
- 間違って生データが公開されないようにフィルターは厳格に

---

## 運用フロー（週次）

```
木〜金: KeibaCICD でレース分析・予測実行
金夜:   Claude API でキャラ別スクリプト生成
金夜:   public_export.py で公開用データ生成
金夜:   Noteに記事投稿 + Webページ自動デプロイ
土日:   レース本番
日夜:   結果反映 → 成績更新 → 結果回スクリプト生成
月:     結果回を投稿
```
