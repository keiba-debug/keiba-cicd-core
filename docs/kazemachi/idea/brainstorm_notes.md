# ブレスト議事録

## Session 113（2026-03-20）— 初回ブレスト

### 議論のハイライト

#### キャラ対立構造
- 「データの種類で分ける」より**予想哲学の対立**で分ける方が議論が熱くなる
- 採用: **データ派 vs 直感派 vs 経験派**
- 三角関係: 全ペアに固有の対立理由がある
  - データ vs 直感 → 「根拠を出せ」「数字じゃわからない」
  - 直感 vs 経験 → 同族嫌悪「お前のは思い出補正」
  - 経験 vs データ → 世代間緊張「昔はそんな数字なかった」

#### キャラの弱点
- 予想の弱さだけでなく**人間的な欠陥**が大事
- 候補: 流されやすい、自信がない、馬名間違える、脱線する、贔屓が入る、G1でビビる
- 弱点があるからこそ視聴者が愛着を持てる

#### 進行役
- アレクシア（ハンドイットオーバー）型
  - 冷静に容赦なくツッコむ
  - ボケ（予想家）を引き立てる最強の聞き手
  - 呆れてるけど見捨てない
- **予想しないけど実は一番詳しい？** という謎を長期的に引っ張る ← 最高の仕掛け

#### バックストーリー設計
- 毎回少しずつ断片的に明かす（ドカンと出さない）
- 3層構造: 表層（毎回）→ 中層（数回に1回）→ 深層（10〜20回かけて）
- キャラ同士に隠された過去の繋がり
- 同じレースの記憶を別の回で別キャラが語る → 合わせると1つの物語

#### 人数
- 1回の出演: 3人＋進行役
- 将来的に人気でローテーション、マンネリ防止

#### データ権利
- IDM・調教コメント等は直接出せない → キャラの言葉に変換
- むしろキャラの個性を強める方向に働く

#### KeibaCICD = キャラの成長
- モデルのバージョンアップがキャラの成長エピソードになる
- 視聴者は気づかないけど裏では実際にMLが改善してる（最高のメタ構造）

## Session 113 追加ブレスト（2026-03-20）— 予想バトルの本質

### 核心の問題提起
- 各AI案はストーリー・バックストーリーが充実しすぎて**ドラマ寄りになりすぎ**
- 本質は「レース前の予想バトル番組」。結果はまだわからない
- **予想バトルが主菜、バックストーリーはスパイス**

### 予想バトルが面白くなる構造（結果に依存しない）
- 「なぜその馬か」の説得バトル → 根拠のぶつかり合い自体が面白い
- 人気馬を「消す」瞬間の緊張感 → 「お前正気か？」
- 全員一致の気まずさ → 「逆に怖くない？」
- 前回の結果フィードバック → 外した人がイジられる → 次の予想への伏線

### バックストーリーの重さ調整
- 今の案: 「15年前のあの日…」「涙」「トラウマ」→ **重すぎ**
- 調整: 「あー、この騎手か。昔ちょっとな…」（それ以上言わない）程度
- 会話の端々に**1回だけチラッと匂わせる**レベル

### 1回分の構成イメージ（実用版・約12分）
```
[0:00] OP — 進行役がレース概要
[0:30] 前回振り返り — 成績発表＋イジり
[1:30] レース概要 — コース特性、注目馬
[3:00] 本命発表 — 3人が順番に本命と理由
[5:00] バトル — 互いの予想にツッコミ・質問
[8:00] 穴馬・危険な人気馬 — 一番盛り上がるパート
[10:00] 最終予想まとめ — 進行役が整理
[11:00] ED — 「来週、誰が笑ってるでしょうね」
```

### 方針
- まずプロトタイプで遊びながら育てる
- 画像生成・動画生成は今後やりやすくなるので焦らない
- キャラは実際に回しながら磨いていく
