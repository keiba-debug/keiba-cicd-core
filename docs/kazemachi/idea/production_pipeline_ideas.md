# 制作パイプライン — 段階別アプローチ

> 完璧を目指さず、今できる形で出す。技術は後から追いつく。

---

## 段階別ロードマップ

### Level 0: テキスト（今すぐ）
- Claude APIでキャラ別発言を生成 → スクリプト組み立て
- HTML/Markdownで出力
- **これだけで内容の面白さは検証できる**
- YouTube コミュニティ投稿やブログでもOK

### Level 1: 紙芝居（少しの準備で）
- キャラ立ち絵を画像生成AIで作成（1キャラ数パターン）
- 立ち絵 + セリフ字幕 + BGM → FFmpeg/MoviePyで結合
- ラジオ的な視聴体験としては十分
- ツール: Python + Pillow + FFmpeg

### Level 2: 音声付き紙芝居
- VOICEVOXでキャラ別音声を生成
- 立ち絵 + 音声 + 字幕 → 動画化
- キャラの声が付くだけで一気にコンテンツ感が出る
- ツール: VOICEVOX + FFmpeg

### Level 3: 表情差分付き
- 立ち絵を表情パターン別に用意（通常/怒り/驚き/落胆/歓喜）
- セリフの感情に合わせて差分を切り替え
- 簡単なアニメーション（口パク等）
- ツール: Live2D or AI表情生成

### Level 4: フル動画（将来）
- AI動画生成でキャラが動く
- リップシンク、ジェスチャー
- レース実況シーンの再現
- ツール: Kling/Runway/Pika等（急速進化中、半年後に再評価）

---

## 現実的な初手

**Level 0 → Level 1 を目指す**

1. まずLevel 0でスクリプトの面白さを確認（自分で楽しむ）
2. 立ち絵を何パターンか生成して雰囲気を見る
3. Level 1で「観れる」最低限の動画にする
4. 音声は後回し（テロップで十分伝わる）

---

## AI動画生成の現状メモ（2026-03時点）

- **画像生成**: Stable Diffusion / DALL-E / Midjourney でキャラ立ち絵は十分作れる
  - 一貫性（同じキャラを毎回同じ顔で出す）が課題 → LoRA/IP-Adapterで対応可
- **音声生成**: VOICEVOX（無料・日本語強い）、Style-Bert-VITS2（高品質）
- **動画生成**: Kling、Runway、Pika — 急速進化中だがまだ制御が難しい
  - 「立ち絵から口パク動画」は技術的には可能になりつつある
  - 安定した品質で毎週量産するにはまだ早い
- **結論**: 技術が追いつくのを待ちながら、中身（スクリプト）を磨くのが正解

## 制作効率の観点

毎週1本出すなら自動化が命：

```
[自動化すべき部分]
- KeibaCICD → レースデータJSON出力
- Claude API → キャラ発言生成
- スクリプト組み立て
- （Level 1以降）画像+字幕+BGM結合

[人間がやる部分]
- スクリプトの最終チェック・微調整
- 面白さの確認（AIだけだとツッコミが甘くなりがち）
- 公開判断
```

週のルーティンに組み込めるレベルまで自動化できれば継続できる。
