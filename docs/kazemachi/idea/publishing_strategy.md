# 公開戦略 — テキストから動画へ段階的に

---

## ステップ

### Step 1: テキストで公開して反応を見る

| プラットフォーム | メリット | デメリット |
|-----------------|---------|-----------|
| **Note** | 競馬ファン層がいる、マガジン機能、有料化可能 | デザインの自由度は低い |
| **Azure Static Web Apps** | 自由度高い、自動化しやすい、自前ブランド | 集客は自力 |
| **X/Twitter** | 拡散力、短縮版で導線作り | 長文には向かない |

**推奨**: Note + X の組み合わせで始める
- Note: 本編（フル会話スクリプト+キャラアイコン）
- X: ハイライト（「今週のバトル：データ派 vs 直感派、本命が真っ二つ」）

### Step 2: ビジュアル付きWebサイト

- Azure Static Web Apps or Vercel
- キャラ立ち絵 + セリフ吹き出し + データカード
- keiba-v2/webと同じNext.js/React技術スタック
- KeibaCICDパイプラインに組み込んで自動デプロイ

### Step 3: 動画化 → YouTube

- 音声+映像が揃ってからYouTubeに移行
- それまではNote/Webで読者を育てておく
- YouTube開始時に「テキスト版から応援してくれてた人」がいる状態が理想

## Note活用の具体案

### マガジン構成
```
📚 予想TV マガジン
  ├── 【予想回】第1話 — ◯◯賞、3人の予想が割れた
  ├── 【結果回】第1話 — 勝ったのは直感派、データ派は沈黙
  ├── 【予想回】第2話 — ...
  └── 【月間成績】3月 — データ派が首位奪還
```

### 記事フォーマット案
```
🏇 予想TV 第X話 — 「◯◯賞」

[キャラアイコン] データ派: 「この馬、メンバー中で能力は頭一つ抜けてる」
[キャラアイコン] 直感派: 「いや、今回はなんか違う。2番の方が来る」
[キャラアイコン] 経験派: 「お前ら、3年前の同条件覚えてないのか」
[キャラアイコン] 進行役: 「で、3人とも買うの？買わないの？」
...

📊 今回の買い目
  データ派: 複勝 3番 500円
  直感派: 単勝 2番 1000円
  経験派: 馬連 2-5 300円

💰 累計成績
  データ派: +1,200円 | 直感派: -800円 | 経験派: +400円
```

## Azure Static Web Apps 構成案

```
yosou-tv-web/
  ├── src/
  │   ├── pages/
  │   │   ├── index.tsx        ← 最新回 + アーカイブ
  │   │   ├── episode/[id].tsx ← 各回の詳細
  │   │   └── standings.tsx    ← キャラ別成績表
  │   ├── components/
  │   │   ├── CharacterLine.tsx ← キャラアイコン+セリフ吹き出し
  │   │   ├── BetCard.tsx      ← 買い目カード
  │   │   └── StandingsTable.tsx
  │   └── data/
  │       └── episodes/        ← 自動生成されたスクリプトJSON
  └── staticwebapp.config.json
```

技術的にはkeiba-v2/webの延長で作れる。

## データ権利の再確認（公開時に重要）

テキストでもWebでも公開する以上、以下は厳守:
- IDM等の指数値は出さない → 「メンバー中トップクラス」等の表現
- 調教コメント原文は使わない → キャラの言葉で表現
- JRDBの生データは表示しない
- レース映像は使わない（結果回はテキスト+リアクションのみ）
- オッズは公開情報なので使用OK
