# AI協調開発 会話ログ

> カカシ、ふくだ君、Cursor による非同期協調開発の記録

---

## 📋 このフォルダについて

**目的**: 複数のAIエージェントによる開発の意思決定・実装・レビューを記録

**原則**:
- すべての開発タスクは conversations/ 配下に記録する
- 非同期コミュニケーションで各自のペースで作業
- 履歴を残し、振り返りと改善を促進

---

## 📂 フォルダ構造

```
conversations/

│   ├── 00_plan.md                       # カカシ: 設計
│   ├── 01_instruction.md                # ふくだ君: 指示
│   ├── 02_implementation.md             # Cursor: 実装報告
│   ├── 03_review.md                     # カカシ: レビュー
│   └── 90_summary.md                    # カカシ: サマリ
├── 2026-02-10_training-db-setup/        # タスク2
│   └── ...
└── README.md                            # このファイル
```

---

## 🚀 クイックスタート

### 新しいタスクを始める

1. **フォルダ作成**
   ```bash
   mkdir conversations/2026-02-XX_{タスク名}
   ```

2. **カカシがプラン作成**
   - `00_plan.md` に設計・方針を記載

3. **ふくだ君が指示作成**
   - `01_instruction.md` にCursor向け実装指示を記載

4. **Cursorが実装**
   - `02_implementation.md` に実装結果を報告

5. **カカシがレビュー**
   - `03_review.md` にレビュー結果を記載

6. **完了時サマリ**
   - `90_summary.md` に振り返りを記載

---

## 📝 ファイル命名規則

### フォルダ名
- フォーマット: `YYYY-MM-DD_{タスク名}`
- タスク名: ケバブケース（小文字、ハイフン区切り）
- 例: `2026-02-07_v3.1-swr-introduction`

### ファイル名
- フォーマット: `{連番}_{種類}.md`
- 連番:
  - `00-09`: プラン・指示
  - `10-19`: 実装報告
  - `20-29`: レビュー
  - `90-99`: サマリ・振り返り
