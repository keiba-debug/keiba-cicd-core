# gunner プレイブック（Fuku↔AIチーム連携）

このファイルは、Fuku（依頼者）とAIエージェント群の間に立つ「gunner」の運用手順テンプレです。

## 0. 前提

- 参照すべきガイド: `ai-team/knowledge/CLAUDE.md` / `ai-team/experts/TEAM.md`
- 目的: **当てる**（期待値と資金管理を含む）ための意思決定を支援する

## 1. 依頼受付（最初に確認すること）

最低限、次を埋める（不明なら質問して埋める）:

- **日付**: 例）2025-12-20
- **対象**: 例）「中山11R」「重賞だけ」「全36レース」など
- **データ場所**: 例）`Z:\KEIBA-CICD\data\organized\...`
- **出力先**: 例）`Z:\KEIBA-CICD\reports\{担当者}\...`
- **締切/優先度**: 例）「発走30分前」「今日中」など

## 2. 役割分担（どのエージェントに投げるか）

迷ったら、まずはこの最小セット:

- **ANALYST**: 統合（各分析の整合・品質チェック）
- **STRATEGIST**: 展開/ペース/隊列（前崩れ・前残りの前提）
- **EVALUATOR**: 馬の能力/適性/状態（上限と下限）
- **INVESTOR**: 券種/点数/資金配分（期待値とリスク）
- **GUARDIAN**: 見送り判断/最大損失/禁止事項（破滅回避）

## 3. 指示プロンプト（コピペ用テンプレ）

### 3.1 ANALYST（統合）

```
対象: {日付} {競馬場}{R} {レース名}
入力: {データパス}
目的: 各分析（展開/馬質/市場）を統合し、結論と根拠を箇条書きで出す。
出力:
- 推奨馬（◎○▲△）と根拠（最大5点）
- 見送り条件（何が起きたら買わないか）
- 投資推奨度（S/A/B/C/見送り）
```

### 3.2 STRATEGIST（展開）

```
対象: {日付} {競馬場}{R}
入力: 出走馬/脚質/枠/想定ペース
出力:
- 想定隊列（逃げ候補・先行群・差し群）
- ペース判定（S/M/H）と根拠
- 有利/不利になりやすい馬タイプ
```

### 3.3 EVALUATOR（馬質）

```
対象: {日付} {競馬場}{R}
入力: 近走、条件替わり、調教、適性（芝ダ/距離/馬場）
出力:
- 能力上位3頭（理由）
- 取りこぼしやすい人気馬（理由）
- 穴で拾うなら（条件一致・上振れ要因）
```

### 3.4 INVESTOR（買い目）

```
対象: {日付} {競馬場}{R}
入力: ◎○▲△、想定オッズ、勝率の相対序列
制約: 点数は最小（例: 馬単1点 / 3連単5点以内 / ワイド1点）
出力:
- 券種別の推奨（点数・買い目）
- 見送り判断基準（オッズ条件など）
```

### 3.5 GUARDIAN（リスク）

```
対象: {日付} {競馬場}{R}
入力: 最終買い目案、資金制約、直前情報
出力:
- NG条件（買ってはいけない状態）
- 最大投資上限（目安）
- 直前チェックリスト（3〜7項目）
```

## 4. 最終統合（gunnerとしてFukuに返す形式）

- **結論**: 買う/見送り
- **買う場合**: 券種・点数・買い目（先にここ）
- **根拠**: 3〜7点
- **懸念**: 1〜3点
- **直前に見る条件**: オッズ/馬場/取消/パドック等
