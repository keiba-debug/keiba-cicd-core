# TARGET frontier JV データ構造一覧

JRA-VAN Data Lab.のデータをTARGET frontier JVが管理するフォルダ構造を解説します。

## ベースパス

デフォルト: `Y:/` (環境変数 `JV_DATA_ROOT_DIR` で変更可能)

## 主要データディレクトリ

| ディレクトリ | 内容 | レコードタイプ |
|-------------|------|---------------|
| DE_DATA | 出馬表データ、開催情報 | DR, RA |
| SE_DATA | 成績データ、配当データ | SE |
| UM_DATA | **馬データ**、産駒データ | UM, SK |
| ES_DATA | 地方・海外成績 | ES |
| HY_DATA | 票数データ、オッズ | HY |
| KT_DATA | 種牡馬・繁殖牝馬 | KT |
| BR_DATA | 生産者データ | BR |
| OW_DATA | 馬主データ | OW |
| CK_DATA | 坂路調教データ | CK |
| RT_DATA | 速報データ (当日) | RT |
| MY_DATA | 買い目・コメント | (ユーザー定義) |

## ファイル命名規則

### 年次/半期分割パターン

```
{レコードコード}{年}{半期}.DAT
{レコードコード}{年}{半期}.IDX
```

- 半期: 1=上半期(1-6月), 2=下半期(7-12月)
- 例: `UM20242.DAT` = 2024年下半期の馬データ

### 日次パターン

```
{レコードコード}{年月日}.DAT
```

- 例: `DR20260124.DAT` = 2026年1月24日の出馬表データ

## レコード形式

- **固定長レコード**: 各レコードタイプごとにバイト長が固定
- **エンコーディング**: Shift-JIS
- **改行コード**: CRLF

## 主なレコード長

| レコードタイプ | 長さ (bytes) | 説明 |
|---------------|-------------|------|
| UM (馬) | 1,609 | 競走馬マスタ |
| RA (レース) | ~751 | レース詳細 |
| SE (成績) | 可変 | レース成績 |
